# Plan 49-03: Milestone Audit, Gap Closure, and Main Loop Wiring

## Objective

Add milestone audit, gap closure loop, milestone completion, and update the main loop and startup block in `autopilot.mjs` — porting the behavior of bash `run_milestone_audit`, `run_gap_closure_loop`, `run_milestone_completion`, and `print_escalation_report`. Also wire `runStepWithRetry` into the execute case and replace the Phase 48 stub comments.

## must_haves

- `runMilestoneAudit` invokes `/gsd:audit-milestone` via `runStepWithRetry`, parses audit file frontmatter, returns 0 (passed/tech-debt-accepted), 10 (gaps_found), or 1 (error)
- `runGapClosureLoop` implements the four-step loop: check iteration limit, plan gaps, execute fix phases through full lifecycle, re-audit
- Gap closure inner loop drives fix phases through discuss-plan-execute-verify-complete with retry and verification gate
- `runMilestoneCompletion` extracts milestone version from STATE.md, invokes `/gsd:complete-milestone` with auto-approve instructions
- `printEscalationReport` displays escalation box when gap closure exhausts iterations
- Execute case in main loop uses `runStepWithRetry` with halt on retry exhaustion
- Complete case triggers milestone audit when `nextIncompletePhase` returns null
- Startup all-phases-complete block triggers milestone audit instead of Phase 48 stub
- All Phase 48 stub comments are removed

<task id="3">
<title>Implement runGapClosureLoop</title>
<actions>
1. Add `async function runGapClosureLoop()`:
   - Read max iterations: `const maxIterations = getConfig('autopilot.max_audit_fix_iterations', 3)`
   - Initialize `iteration = 0`
   - While loop:
     a. Check iteration limit BEFORE planning: if `iteration >= maxIterations`, call `printEscalationReport(iteration, maxIterations)` and `process.exit(1)`
     b. Increment iteration
     c. Log: `GAP CLOSURE: iteration=${iteration}/${maxIterations}`
     d. Print banner: `GAP CLOSURE: Iteration ${iteration}/${maxIterations}`
     e. Reset circuit breaker: `noProgressCount = 0; iterationLog = []`
     f. Plan gaps: `const gapPlanExit = await runStepWithRetry('/gsd:plan-milestone-gaps --auto', 'gap-planning')`
     g. If gapPlanExit !== 0: print error, call `printEscalationReport`, `process.exit(1)`
     h. Inner fix phase loop (while true):
        - `const fixPhase = findFirstIncompletePhase(PROJECT_DIR)`
        - If no fixPhase: break (all fix phases done)
        - Set `CURRENT_PHASE = fixPhase`
        - Reset: `noProgressCount = 0; iterationLog = []`
        - Inner lifecycle loop (while true):
          - Get step via `getPhaseStep(CURRENT_PHASE)`
          - Switch on step:
            - `discuss`: `await runStep(\`/gsd:discuss-phase ${CURRENT_PHASE} --auto\`, 'discuss')`
            - `plan`: `await runStep(\`/gsd:plan-phase ${CURRENT_PHASE} --auto\`, 'plan')`
            - `execute`: call `runStepWithRetry`, halt on failure
            - `verify`: call `runVerifyWithDebugRetry`, halt on failure; then verification gate (or dry-run auto-approve); then `gsdTools('phase', 'complete', CURRENT_PHASE)`
            - `complete`: print banner `Fix Phase ${CURRENT_PHASE} COMPLETE`, break inner lifecycle loop
            - default: print error, `process.exit(1)`
     i. Print banner: `RE-AUDIT: After iteration ${iteration}`
     j. Call `runMilestoneAudit()`, capture result
     k. If 0: print banner `GAP CLOSURE COMPLETE`, print `Audit passed after ${iteration} iteration(s).`, return
     l. If 10: print `Gaps remain after iteration ${iteration}. Continuing...`, continue loop
     m. Else: print error, `process.exit(1)`

Note: The inner fix loop references `runVerificationGate` from Plan 49-02. Since both plans are wave 2 and modify different function blocks in the same file, the executor should apply 49-03 after 49-02 or ensure the function exists. The `runVerificationGate` function will be defined when this code runs because both plans add to the same file.
</actions>
</task>
