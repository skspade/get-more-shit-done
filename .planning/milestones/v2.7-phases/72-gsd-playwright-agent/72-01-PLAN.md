name: gsd-playwright

**`<role>` section:**
- Identity: GSD Playwright agent — lifecycle agent for E2E testing
- Spawned by: `/gsd:ui-test` command (Phase 73), `add-tests` workflow (Phase 74)
- Core responsibilities: detect Playwright state, scaffold if needed, generate specs from acceptance criteria, execute tests, categorize failures, report structured results
- Include "CRITICAL: Mandatory Initial Read" block for `<files_to_read>`
- Leaf agent constraint: cannot spawn subagents, uses Bash tool directly

**`<input>` section:**
Document the input contract as a table:
| Field | Source | Purpose |
|-------|--------|---------|
| mode | Task() prompt | `ui-test` (full lifecycle), `generate` (skip detect/scaffold), `scaffold` (scaffold only) |
| phase_dir | Task() prompt | Path to phase directory for reading CONTEXT.md |
| base_url | Task() prompt | Application URL (default: http://localhost:3000) |
| flags | Task() prompt | `--scaffold` (force scaffold), `--run-only` (skip generation), `--headed` (visible browser) |

**`<process>` section — Steps 1-2:**

**Step 1: Detect Playwright State**
- Skip if mode is `generate` (assume configured)
- Run: `node get-shit-done/bin/gsd-tools.cjs playwright-detect`
- Parse JSON result for status: configured, installed, not-detected
- If `configured` and no `--scaffold` flag: skip to Step 3 (or Step 5 if mode is `scaffold`)
- If `installed`: proceed to scaffold (skip npm install, create config and browser install only)
- If `not-detected`: proceed to full scaffold

**Step 2: Scaffold Playwright**
- Skip if detection returned `configured` (unless `--scaffold` flag)
- If `not-detected`:
  1. Run `npm install -D @playwright/test`
  2. Create `playwright.config.ts` using the config template from CONTEXT.md (testDir: './e2e', fullyParallel: true, reporter: 'line', screenshot: 'only-on-failure', trace: 'on-first-retry', Chromium-only project, baseURL from input)
  3. Create `e2e/` directory with `example.spec.ts` smoke test (imports test/expect from @playwright/test, navigates to baseURL, checks page title is not empty)
  4. Append to `.gitignore`: `test-results/`, `playwright-report/`, `blob-report/`, `.playwright/` (only if not already present)
  5. Run `npx playwright install chromium`
- If `installed` (dep exists, no config): skip npm install, do steps 2-5
- If mode is `scaffold`: report and stop (skip Steps 3-4)
- retries: `process.env.CI ? 2 : 0`, workers: `process.env.CI ? 1 : undefined`

<task type="auto">
  <name>Task 2: Add generate, execute, and report steps with failure categorization and structured return</name>
  <files>agents/gsd-playwright.md</files>
  <action>
Continue building the `<process>` section and add the `<output>` section:

**Step 3: Generate Test Specs**
- Skip if `--run-only` flag
- Read `{phase_dir}/*-CONTEXT.md` using Read tool
- Look for `<acceptance_tests>` section
- If no `<acceptance_tests>` section found: set status to BLOCKED with reason "No acceptance tests defined" and skip to Step 5
- For each acceptance criterion with Given/When/Then/Verify:
  - Map to a `test()` block inside a `test.describe()` group
  - Use locator priority hierarchy: `getByRole` > `getByText` > `getByLabel` > `getByTestId` > CSS selector (last resort)
  - Given → test setup (navigation, state preparation)
  - When → user actions (clicks, fills, navigates)
  - Then/Verify → assertions (expect statements)
- Write generated file to `e2e/{phase-slug}.spec.ts`
- Import `{ test, expect }` from `@playwright/test`

**Step 4: Execute Tests**
- Skip if mode is `scaffold`
- Run: `npx playwright test` (add `--headed` if flag set, add `--project=chromium`)
- Capture stdout and stderr from Bash tool output
- Parse results using regex patterns matching testing.cjs:
  - Pass count: `/(\d+)\s+passed/`
  - Fail count: `/(\d+)\s+failed/`
  - Skip count: `/(\d+)\s+skipped/`
  - Failed test names: `/^\s+\d+\)\s+(.+)$/gm`
- If Bash exit code is non-zero and output is empty: application-level failure

**Step 5: Categorize Failures and Report**

**Failure categorization** (only when tests fail):
For each failure, classify by error message pattern:
- **Application-level** (app not running or broken): `ERR_CONNECTION_REFUSED`, `net::ERR_`, `ECONNREFUSED`, `NS_ERROR_CONNECTION_REFUSED`, `page.goto: Timeout`, `Navigation timeout`
- **Test-level** (test needs adjustment): `locator.`, `not found`, `strict mode violation`, `expect(received)`, `toHaveText`, `toBe`, `toBeVisible`, assertion-related keywords

Collect screenshot paths from `test-results/` directory using Glob.
Collect trace file paths from `test-results/` directory (files ending in `.zip`).

## PLAYWRIGHT COMPLETE
