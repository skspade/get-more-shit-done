**2. Scaffolding Prompt (WKFL-02):**
When Playwright is not configured, present an AskUserQuestion:
```
AskUserQuestion(
  header: "Playwright Not Detected",
  question: "Playwright is not configured in this project. E2E test generation requires Playwright. How would you like to proceed?",
  options:
    - "Scaffold Playwright and continue" — creates playwright.config.ts, e2e/ directory, installs Chromium
    - "Skip E2E tests" — bypass E2E generation, record zeros in summary
    - "Cancel" — stop add-tests workflow
)
```

If user chooses "Scaffold Playwright and continue", execute inline scaffolding mirroring gsd-playwright.md Step 2:
- If status was `not-detected`: run `npm install -D @playwright/test`
- Create `playwright.config.ts` with: testDir `./e2e`, fullyParallel true, reporter `line`, chromium-only project, screenshot only-on-failure, trace on-first-retry, baseURL `http://localhost:3000`
- Create `e2e/` directory with `e2e/example.spec.ts` smoke test (imports from @playwright/test, goes to /, asserts title matches /.+/)
- Read `.gitignore`, append `test-results/`, `playwright-report/`, `blob-report/`, `.playwright/` if not already present
- Run `npx playwright install chromium`

If user chooses "Skip E2E tests": set E2E counts to zero (generated=0, passing=0, failing=0, blocked=0), skip to summary step.
If user chooses "Cancel": exit workflow.

**3. Playwright Spec Generation (WKFL-03):**
For each approved E2E test (from the classification step):

a. Check for existing tests covering the same scenario:
```bash
grep -r "{scenario keyword}" e2e/ 2>/dev/null
```
If found, extend rather than duplicate.

b. Create spec file at `e2e/{feature-slug}.spec.ts` with:
- Import `{ test, expect }` from `@playwright/test`
- `test.describe('{Feature Name}', () => { ... })` groups from acceptance criteria categories
- Each acceptance criterion maps to a `test()` block:
  - Given section maps to test setup (navigation, state preparation)
  - When section maps to user actions (clicks, fills, navigates)
  - Then/Verify section maps to assertions (expect statements)
- Locator priority hierarchy: `page.getByRole()` > `page.getByText()` > `page.getByLabel()` > `page.getByTestId()` > CSS selectors

**4. RED-GREEN Execution (WKFL-04):**
Run each generated spec:
```bash
npx playwright test --project=chromium {spec-file}
```

Evaluate results:
- **GREEN (passes)**: Record success
- **RED (fails)**: Categorize using gsd-playwright error patterns:
  - App-level (flag as blocker, do NOT fix): ERR_CONNECTION_REFUSED, net::ERR_, ECONNREFUSED, page.goto: Timeout, Navigation timeout, Target page/context/browser closed
  - Test-level (fix the test, re-run): locator., strict mode violation, expect(received), Expected, Received, toHaveText, toBe, toBeVisible, toHaveCount, not found, element not visible
- **Cannot run**: Report blocker. Do NOT mark as complete.

Preserve the no-skip rule: If E2E tests cannot execute, report the blocker and mark as incomplete.
