<feature>
  <name>cmdTestRun -- test execution with baseline comparison and output summarization</name>
  <files>get-shit-done/bin/lib/testing.cjs, get-shit-done/bin/gsd-tools.cjs, tests/testing.test.cjs</files>
  <behavior>
    ## Core execution
    - `cmdTestRun(cwd, { baseline: false, baselineData: null, commitMsg: null }, raw)` runs the test command resolved via `getTestConfig(cwd)` and returns structured JSON
    - Return shape: `{ status: 'pass'|'fail'|'skip'|'error', total, passed, failed, new_failures: [], baseline_failures: [], summary: string, raw_length: number }`
    - When test command exits 0: status='pass', failed=0, new_failures=[]
    - When test command exits non-zero: status='fail', with parsed failure counts and test names
    - When no test command available (no framework detected, no config): status='skip', summary includes reason

    ## Baseline capture
    - `cmdTestRun(cwd, { baseline: true }, raw)` runs tests and returns baseline data structure
    - Baseline return shape adds `baseline` field: `{ ..., baseline: { exitCode, total, passed, failed, failedTests: string[] } }`
    - Baseline captures the current state of test failures for later comparison

    ## Baseline comparison
    - When `baselineData` is provided, compare current `failedTests` against `baselineData.failedTests`
    - `new_failures` = tests in current failures NOT in baseline failures
    - `baseline_failures` = tests that were already failing in baseline
    - If `new_failures.length === 0` even though tests fail: status='pass' (only pre-existing failures)
    - If `new_failures.length > 0`: status='fail'

    ## TDD RED detection
    - When `commitMsg` starts with `test(`: status='skip', summary='TDD RED commit detected -- gate skipped'
    - Detection happens BEFORE running any tests (short-circuit)

    ## Output summarization
    - Parse test runner output for counts using framework-specific patterns
    - node:test (TAP): look for `# tests N`, `# pass N`, `# fail N`, `not ok N - description`
    - jest: look for `Tests:\s+(\d+) failed,\s+(\d+) passed,\s+(\d+) total`
    - vitest: look for `Tests\s+(\d+) failed \| (\d+) passed`
    - mocha: look for `(\d+) passing`, `(\d+) failing`
    - Fallback: exit code 0 = pass (unknown counts), non-zero = fail (unknown counts)
    - `raw_length` reports byte length of captured output (NOT included in return)

    ## Hard gate config check
    - When `getTestConfig(cwd).hard_gate` is false: status='skip', summary='Hard gate disabled'
    - Check happens before test execution

    ## Error handling
    - execSync timeout (default 120s): status='error', summary includes timeout info
    - Other execution errors: status='error', summary includes error message

    ## Dispatcher integration
    - `gsd-tools.cjs test-run` routes to `testing.cmdTestRun(cwd, options, raw)`
    - Supports flags: `--baseline`, `--baseline-data <json>`, `--commit-msg <msg>`
  </behavior>
  <implementation>
    1. Add internal helper `parseTestOutput(stdout, stderr, framework)` that extracts counts and failing test names using regex patterns per framework. Returns `{ total, passed, failed, failedTests: string[] }`.

    2. Add internal helper `runTests(cwd, command, timeout)` that calls `execSync(command, { cwd, stdio: ['pipe','pipe','pipe'], timeout, encoding: 'utf-8' })`. Catches errors and extracts stdout/stderr from the error object. Returns `{ exitCode, stdout, stderr }`.

    3. Add exported function `cmdTestRun(cwd, options, raw)`:
       - Check hard_gate config -- if false, output skip and return
       - Check commitMsg for TDD RED pattern (`/^test\(/`) -- if match, output skip and return
       - Get test command from `getTestConfig(cwd)` -- if none, output skip and return
       - Call `runTests(cwd, command, 120000)`
       - Parse output via `parseTestOutput(stdout, stderr, framework)`
       - If baseline mode: include baseline data in output
       - If baselineData provided: compute new_failures via Set difference
       - Build summary string from parsed results
       - Call `output(result, raw, result.summary)`

    4. Add `test-run` case to gsd-tools.cjs dispatcher (before the `default:` case):

       ```javascript
         testing.cmdTestRun(cwd, { baseline: baselineFlag, baselineData, commitMsg }, raw);
         break;
       }
       ```

    5. Add `cmdTestRun` to module.exports in testing.cjs.
