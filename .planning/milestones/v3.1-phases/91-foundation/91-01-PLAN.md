<feature>
  <name>UAT Config Loading and Validation</name>
  <files>get-shit-done/bin/lib/uat.cjs, get-shit-done/bin/lib/uat.test.cjs</files>
  <behavior>
    loadUatConfig(planningDir):
    - When .planning/uat-config.yaml does not exist -> returns null
    - When config has valid base_url and no other fields -> returns config with all defaults applied
    - When config has all fields specified -> returns config with exact values
    - When config is missing base_url -> throws Error with descriptive message
    - When config has invalid base_url (no protocol) -> throws Error mentioning "http:// or https://"
    - When config has invalid browser value -> throws Error listing valid options
    - When config has negative startup_wait_seconds -> throws Error
    - When config has negative timeout_minutes -> throws Error

    validateUatConfig(parsed):
    - Validates base_url is present and a valid URL
    - Validates browser is one of "chrome-mcp" or "playwright"
    - Validates fallback_browser is one of "chrome-mcp" or "playwright"
    - Validates startup_wait_seconds is a positive number when provided
    - Validates timeout_minutes is a positive number when provided
    - Applies defaults: startup_wait_seconds=10, browser="chrome-mcp", fallback_browser="playwright", timeout_minutes=10
  </behavior>
  <implementation>
    1. Install js-yaml: `npm install js-yaml`
    2. Create get-shit-done/bin/lib/uat.cjs following the CJS module pattern from core.cjs and testing.cjs:
       - require('js-yaml'), require('fs'), require('path')
       - loadUatConfig(planningDir): check fs.existsSync for uat-config.yaml, return null if missing, otherwise yaml.load + validateUatConfig
       - validateUatConfig(config): validate base_url (required, new URL()), validate browser/fallback_browser enum, validate numeric positivity, apply defaults via nullish coalescing
       - module.exports = { loadUatConfig, validateUatConfig }
  </implementation>
</feature>
