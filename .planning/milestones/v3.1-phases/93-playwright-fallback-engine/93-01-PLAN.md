Replace it with the full Playwright execution engine. The replacement section should instruct the agent to:

**Playwright execution** (when browser_mode is "playwright"):

For each test in the discovered test list:

1. **Timeout check:** Same as Chrome MCP — check elapsed time, mark remaining as `skipped` if timeout exceeded.

2. **Generate ephemeral script:** Write a temporary Node.js file (e.g., `/tmp/uat-test-{N}.cjs`) that:
   - Requires `playwright` and launches headless Chromium: `const { chromium } = require('playwright');`
   - Creates a new browser context and page
   - Navigates to the test URL (base_url + path)
   - Performs interactions based on the test description (the agent translates natural language test steps into Playwright API calls when writing the script):
     - Click actions: `await page.getByText('{text}').click()` or `await page.click('{selector}')`
     - Fill actions: `await page.getByLabel('{label}').fill('{value}')` or `await page.fill('{selector}', '{value}')`
     - Navigation: `await page.goto('{url}')`
     - Keyboard: `await page.keyboard.press('{key}')`
   - Ensures the evidence directory exists: uses `require('fs').mkdirSync('{evidence_dir}', { recursive: true })`
   - Takes a full-page screenshot: `await page.screenshot({ path: '{evidence_path}', fullPage: true })`
   - Extracts page text: `const text = await page.innerText('body')`
   - Outputs JSON to stdout: `console.log(JSON.stringify({ pageText: text, screenshotPath: '{evidence_path}' }))`
   - Closes browser: `await browser.close()`
   - Wraps everything in an async IIFE with try/catch that outputs error JSON on failure

3. **Execute script:** Run via Bash:
   ```bash
   node /tmp/uat-test-{N}.cjs
   ```
   Capture stdout (JSON result) and exit code.

4. **Clean up script:** Delete the temp file:
   ```bash
   rm -f /tmp/uat-test-{N}.cjs
   ```

5. **Parse result:**
   - If exit code is 0 and valid JSON returned: extract `pageText` and `screenshotPath`
   - If exit code is non-zero: record test as failed with the error output as the observed behavior

6. **Judge Pass/Fail:** Apply the SAME judgment protocol as Chrome MCP mode:
   - **PRIMARY signal (DOM text):** Does the pageText contain the expected content?
   - **SUPPLEMENTARY signal (screenshot):** The screenshot was saved to disk for evidence.
   - **Verdict:** PASS if pageText matches expected behavior. FAIL if contradicts or missing.

7. **Record result:** Same format as Chrome MCP:
   - PASS: `{phase, name, status: "pass", evidence: "{path}"}`
   - FAIL: `{phase, name, status: "fail", evidence: "{path}", expected: "{expected}", observed: "{what pageText showed}"}`

The key points to emphasize in the workflow text:
- The agent writes different Playwright code per test based on the test description (same as Chrome MCP where the agent decides which MCP tools to call)
- Scripts use CJS require (not ESM imports) for consistency with the project
- Scripts are ephemeral — written, executed, deleted
- The agent remains the judgment engine — Playwright is just the browser driver
- Output format is identical to Chrome MCP mode
