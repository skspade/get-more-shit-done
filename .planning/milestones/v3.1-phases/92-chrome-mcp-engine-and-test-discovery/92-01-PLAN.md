Create the workflow file at `get-shit-done/workflows/uat-auto.md`. This is a markdown prompt file (not executable code) that a Claude agent follows when spawned via `/gsd:uat-auto`.

The workflow must include these sections in this structure:

**1. Purpose block:** Describe the workflow as an autonomous UAT executor that discovers tests, probes Chrome MCP, executes browser tests, and writes structured results. Emphasize: fully autonomous, no user interaction, no subagent spawning.

**2. Process with sequential steps:**

**Step 1: Load Config**
- Read `.planning/uat-config.yaml` using the Bash tool to run: `node -e "const uat = require('./get-shit-done/bin/lib/uat.cjs'); const cfg = uat.loadUatConfig('.planning'); if (!cfg) { console.log('SKIP'); process.exit(0); } console.log(JSON.stringify(cfg));"`
- If result is "SKIP": write a brief message to stdout ("UAT skipped: no uat-config.yaml found") and exit
- Parse the JSON config object to extract: base_url, startup_command, startup_wait_seconds, browser, fallback_browser, timeout_minutes
- Record the start timestamp

**Step 2: Discover Tests**
- Use Glob to find all `*-UAT.md` files in `.planning/phases/*/`
- For each found file, use Read to check YAML frontmatter for `status: complete`
- From each complete UAT.md, extract test entries (test name and expected behavior description)
- If zero UAT.md files found with status:complete, use Glob to find `*-SUMMARY.md` files across milestone phases
- From each SUMMARY.md, extract key behaviors and success criteria as test scenarios (each becomes a test with name and expected behavior)
- If still zero tests after both sources: write MILESTONE-UAT.md with status:passed, total:0, and a note "No UAT-testable changes found". Commit and exit.
- Build a structured test list: for each test, record phase, test name, expected behavior, and URL hint (inferred from test context or default to base_url)

**Step 3: Detect Browser**
- If config.browser is "chrome-mcp", attempt a full round-trip probe:
  1. Call `mcp__claude-in-chrome__tabs_context_mcp` to check connectivity
  2. If step 1 succeeds, call `mcp__claude-in-chrome__navigate` with the base_url
  3. If step 2 succeeds, call `mcp__claude-in-chrome__get_page_text` to verify content is returned
  4. If all three succeed: use chrome-mcp mode
  5. If any step fails or errors: log the failure, set browser mode to config.fallback_browser
- If config.browser is "playwright": skip probe, use playwright mode directly
- Record the detected browser mode for MILESTONE-UAT.md

**Step 4: Start App (if needed)**
- Use Bash to check if app is running: `curl -s -o /dev/null -w "%{http_code}" {base_url}` (timeout 5s)
- If app responds (any HTTP status): skip startup
- If app does not respond and startup_command is configured: start app via Bash in background (`{startup_command} &`), wait startup_wait_seconds, then re-check with curl
- If app does not respond and no startup_command: abort with error "App not running at {base_url} and no startup_command configured"

**Step 5: Execute Tests**
- Track wall-clock time. Before each test, check if elapsed time exceeds timeout_minutes. If so, mark remaining tests as "skipped" and proceed to Step 6.
- For each test in the discovered test list:

  **Chrome MCP mode:**
  a. Navigate to the test URL using `mcp__claude-in-chrome__navigate`
  b. Perform described interactions using `mcp__claude-in-chrome__computer` (for clicks), `mcp__claude-in-chrome__form_input` (for form fills), or `mcp__claude-in-chrome__shortcuts_execute` (for keyboard actions)
  c. Read DOM text content using `mcp__claude-in-chrome__get_page_text`
  d. Capture screenshot using `mcp__claude-in-chrome__read_page`
  e. Save screenshot to `.planning/uat-evidence/{milestone}/{phase}-test-{N}.png` using Write tool (decode base64 if needed) or Bash
  f. Judge pass/fail:
     - PRIMARY: Compare DOM text content against expected behavior. Does the page contain the expected text, elements, or state described in the test?
     - SUPPLEMENTARY: Use screenshot visual to confirm or add context
     - PASS if: DOM content matches expected behavior description
     - FAIL if: DOM content contradicts expected behavior, expected elements are missing, or error messages are present
  g. For failures: record what was observed vs what was expected

  **Playwright mode:**
  - Note: "Playwright execution engine is implemented in Phase 93. If browser mode is playwright, record all tests as 'skipped' with reason 'Playwright fallback not yet implemented' and set status to passed (no failures detected)."

- Collect results for each test: {phase, name, status (pass/fail/skipped), evidence_path, observed (for failures), expected (for failures)}

**Step 6: Write Results**
- Create `.planning/MILESTONE-UAT.md` using the Write tool with this structure:
  - YAML frontmatter: status (passed if all pass/skip, gaps_found if any fail), milestone (from milestone context), browser (detected mode), started (timestamp), completed (now), total (count), passed (count), failed (count)
  - `## Results` table: | # | Phase | Test | Status | Evidence |
  - `## Gaps` section (only if failures exist): for each failed test, write a gap entry in YAML format in the body with fields: truth (expected behavior), status (failed), reason ("Automated UAT: " + observed description), severity (major), evidence (screenshot path), observed (what was actually seen)
- Create the evidence directory if it does not exist: `mkdir -p .planning/uat-evidence/{milestone}/`

**Step 7: Commit**
- Use Bash to commit results: `git add .planning/MILESTONE-UAT.md .planning/uat-evidence/ && git commit -m "test(uat): automated UAT results for {milestone}"`

**Step 8: Exit**
- Print the status (passed or gaps_found) to stdout
- Exit normally (the spawning process reads MILESTONE-UAT.md frontmatter for the actual status)

**3. Constraints section:**
- No user interaction (WKFL-02)
- No subagent spawning (Task() calls) -- all logic executes in this single agent session
- No modification of source code -- this is read-only testing
- No modification of existing UAT.md files -- read from them, write to MILESTONE-UAT.md
- Timeout enforcement: check wall-clock before each test (WKFL-04)

**4. Chrome MCP Tool Reference** (quick lookup for the agent):
- `mcp__claude-in-chrome__tabs_context_mcp` -- Check Chrome MCP connectivity
- `mcp__claude-in-chrome__tabs_create_mcp` -- Create new browser tab
- `mcp__claude-in-chrome__navigate` -- Navigate to URL
- `mcp__claude-in-chrome__computer` -- Click, type, mouse events
- `mcp__claude-in-chrome__form_input` -- Fill form fields
- `mcp__claude-in-chrome__shortcuts_execute` -- Keyboard shortcuts
- `mcp__claude-in-chrome__read_page` -- Capture page screenshot
- `mcp__claude-in-chrome__get_page_text` -- Extract DOM text content
- `mcp__claude-in-chrome__find` -- Find elements on page

**5. Judgment Protocol** (instructions for the agent on how to judge pass/fail):
- DOM text content is the PRIMARY assertion signal
- Screenshots are SUPPLEMENTARY evidence for debugging
- A test PASSES if: the DOM text content contains the expected elements, text, or state described in the test's expected behavior
- A test FAILS if: the DOM text contradicts the expected behavior, expected elements are absent, error messages appear, or the page is in an unexpected state
- For failures, always record: what was expected (from test description) and what was observed (from DOM text)
- Do NOT accumulate screenshots in conversation context -- save to disk and move on
