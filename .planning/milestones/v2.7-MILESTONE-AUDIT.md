## Tech Debt Summary

**4 items across cross-phase integration:**

1. **INT-01** — `playwright-detect --raw` documented as JSON in 3 consumers but returns plain string. Routing logic works because it matches on the string values directly. Documentation mismatch creates fragility for future modifications.

2. **INT-02** — Phase 72 and 74 SUMMARY frontmatter names `parsePlaywrightOutput()` as a dependency. No such export exists — actual function is `parseTestOutput(stdout, stderr, 'playwright')`. Documentation-only error.

3. **INT-03** — Research explicitly flagged missing `webServer` block as Pitfall 5. Delivered scaffolding in both agent and workflow omits it. Users get `ERR_CONNECTION_REFUSED` if dev server isn't running. Failure categorization correctly catches this as "app-level failure" but could be prevented.

4. **INT-04** — Agent uses relative path (`get-shit-done/bin/gsd-tools.cjs`), command/workflow use absolute (`$HOME/.claude/get-shit-done/bin/gsd-tools.cjs`). Works because agent runs in project context. Different conventions across callers.
