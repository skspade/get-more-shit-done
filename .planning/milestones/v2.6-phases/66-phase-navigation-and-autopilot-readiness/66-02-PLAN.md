Add readiness checks to validation.cjs:

1. Add import at top (if not already present from plan 01):
```javascript
const { CONFIG_DEFAULTS } = require('./config.cjs');
```

2. Add READY-01 check to checks array:
   - id: 'READY-01', category: 'readiness', severity: 'info'
   - Skip if ROADMAP.md missing (return passed: true)
   - Call findFirstIncompletePhase(cwd)
   - If result is not null, pass with message "Incomplete phase found: {phaseNum}"
   - If result is null, fail with message "No incomplete phases found"
   - The check result should include `nextPhase: result` (the phase number or null)

3. Add READY-02 check:
   - id: 'READY-02', category: 'readiness', severity: 'error'
   - Skip if ROADMAP.md missing
   - Call findFirstIncompletePhase(cwd)
   - If no incomplete phase, return passed: true (nothing to validate)
   - Call findPhaseInternal(cwd, incompletePhase) then computePhaseStatus
   - Check that step is in ['discuss', 'plan', 'execute', 'verify']
   - If valid step, pass with message; include `phaseStep: status.step` in result
   - If null/unexpected step, fail with error message
   - Include `nextPhase` and `phaseStep` in the result object

4. Add READY-03 check:
   - id: 'READY-03', category: 'readiness', severity: 'error'
   - Skip if ROADMAP.md missing
   - Call findFirstIncompletePhase(cwd) to find next phase
   - If no incomplete phase, return passed: true
   - Call findPhaseInternal to get the phase directory
   - If no phase dir, return passed: true
   - Read files in the phase directory
   - Check CONTEXT.md files: if any exist and are <= 50 bytes, fail
   - Check PLAN.md files: if any exist and don't contain `<task` (case-insensitive) or `## Task` heading, fail
   - If all artifacts are valid, pass

5. Add READY-04 check:
   - id: 'READY-04', category: 'readiness', severity: 'warning'
   - Skip if config.json missing (return passed: true)
   - Read and parse config.json (if parse fails, return passed: true -- STRUCT-02 handles parse errors)
   - If no `autopilot` key in config, pass
   - Extract known autopilot subkeys from CONFIG_DEFAULTS: filter keys starting with 'autopilot.', map to subkey after the dot
   - Check each key in config.autopilot against known subkeys; collect unknown keys
   - Check numeric values (circuit_breaker_threshold, max_debug_retries, max_audit_fix_iterations, stall_timeout_ms) are positive integers when present
   - If any issues, fail with message listing them
   - Otherwise pass

6. Update validateProjectHealth to populate nextPhase and phaseStep:
   - After running checks, scan results for entries that have `nextPhase` or `phaseStep` properties
   - Set `nextPhase` and `phaseStep` on the return object from the first result that provides them
   - This requires a small addition after `const results = runChecks(...)`:
   ```javascript
   let nextPhase = null;
   let phaseStep = null;
   for (const r of results) {
     if (r.nextPhase !== undefined && nextPhase === null) nextPhase = r.nextPhase;
     if (r.phaseStep !== undefined && phaseStep === null) phaseStep = r.phaseStep;
   }
   ```

7. Update runChecks to pass through extra properties from check results:
   - Currently runChecks creates result objects with specific fields. Add spread of any extra fields from the check result:
   ```javascript
   const result = entry.check(cwd);
   const extraFields = {};
   if (result.nextPhase !== undefined) extraFields.nextPhase = result.nextPhase;
   if (result.phaseStep !== undefined) extraFields.phaseStep = result.phaseStep;
   results.push({
     id: entry.id,
     category: entry.category,
     severity: result.severity || entry.severity,
     passed: result.passed,
     message: result.message,
     repairable: typeof entry.repair === 'function',
     repairAction: null,
     ...extraFields,
   });
   ```

Key constraints:

- Catch all errors internally, return { passed, message }
- READY-03 only checks artifacts in the NEXT incomplete phase, not all phases
