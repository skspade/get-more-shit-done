Create `get-shit-done/bin/lib/validation.cjs` following CJS module conventions from existing lib modules.

Module structure:
```
/**
 * Validation — Project health check registry and validation engine
 */

const fs = require('fs');
const path = require('path');
```

1. **Check Registry** — Module-level array of check objects:
   ```
   const checks = [
     {
       id: 'STRUCT-01',
       category: 'structure',
       severity: 'error',
       check: (cwd) => {
         const exists = fs.existsSync(path.join(cwd, '.planning'));
         return { passed: exists, message: exists ? '.planning/ directory exists' : '.planning/ directory not found' };
       },
     },
   ];
   ```

2. **runChecks(cwd, options)** — Execute checks with optional category filtering:
   - Accept `{ categories }` in options
   - Filter checks array by category when categories provided
   - Execute each check's `check(cwd)` function
   - Map results to CheckResult shape: `{ id, category, severity, passed, message, repairable, repairAction }`
   - `repairable` is `typeof entry.repair === 'function'`
   - `repairAction` is `null` (repairs not wired this phase)

3. **validateProjectHealth(cwd, options)** — Primary entry point:
   - Accept `{ categories, autoRepair }` in options (autoRepair accepted but not implemented)
   - Call `runChecks(cwd, { categories })`
   - Partition results: errors = failed + severity 'error', warnings = failed + severity 'warning'
   - `healthy` = `errors.length === 0`
   - Return `{ healthy, checks, errors, warnings, repairs: [], nextPhase: null, phaseStep: null }`

4. **Exports** — `module.exports = { validateProjectHealth, runChecks };`

Key constraints:
- NO console output, NO ANSI codes — return data only
- NO imports from verify.cjs — one-way dependency
- NO eager imports of state.cjs — lazy only (not needed this phase)
- NO classes or EventEmitter — plain functions and arrays
- Synchronous execution only (matching project convention)
