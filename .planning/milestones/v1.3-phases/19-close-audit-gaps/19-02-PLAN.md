### What to do

Add a new integration test in the `gsd-cli binary` describe block in `tests/cli.test.cjs`. The test should call the CLI binary with `todos --area=<value> --json` and verify the filtering works.

Since the `handleTodos` function parses `--area` from `process.argv` (not from `parseArgs`), this integration test calling the binary is the only way to exercise this code path end-to-end.

**Add this test after the existing `todos --json` integration test (after line 257):**

```javascript
  test('todos --area filters by area (TODO-02)', () => {
    // Use a temp directory with controlled todo fixtures to avoid depending on project state
    const tmpDir = fs.mkdtempSync(path.join(os.tmpdir(), 'gsd-area-test-'));
    const pendingDir = path.join(tmpDir, '.planning', 'todos', 'pending');
    fs.mkdirSync(pendingDir, { recursive: true });
    // Create two todos with different areas
    fs.writeFileSync(path.join(pendingDir, 'bug-one.md'),
      '---\ntitle: Bug One\narea: bugfix\ncreated: 2026-01-01\n---\nBug details\n');
    fs.writeFileSync(path.join(pendingDir, 'feat-one.md'),
      '---\ntitle: Feature One\narea: feature\ncreated: 2026-01-02\n---\nFeature details\n');
    try {
      const output = execSync(`node "${cliPath}" todos --area=bugfix --json`, { cwd: tmpDir, encoding: 'utf-8' });
      const parsed = JSON.parse(output);
      assert.strictEqual(parsed.command, 'todos');
      assert.strictEqual(parsed.count, 1);
      assert.ok(Array.isArray(parsed.todos));
      for (const todo of parsed.todos) {
        assert.strictEqual(todo.area, 'bugfix', 'Every returned todo should have area=bugfix');
      }
    } finally {
      fs.rmSync(tmpDir, { recursive: true, force: true });
    }
  });
```

This test:
1. Creates a temp directory with two todos in different areas (bugfix and feature)
2. Calls the CLI binary with `todos --area=bugfix --json` pointing at the temp directory
3. Asserts `command` is `"todos"`, `count` is 1, and every todo has `area: "bugfix"`
4. Cleans up the temp directory
