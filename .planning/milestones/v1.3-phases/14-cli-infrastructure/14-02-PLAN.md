# Plan 02: Command Routing and Stub Handlers

## Objective

Add command routing to the CLI entry point so that `gsd progress`, `gsd todos`, `gsd health`, `gsd settings`, and `gsd help` dispatch to stub handler functions. Unknown commands show an error listing available commands. Bare `gsd` invocation shows the help output. This completes CLI-03 and the full Phase 14 scope.

## Must-Haves (Goal-Backward)

- `gsd progress` dispatches to progress handler stub (returns placeholder output)
- `gsd todos` dispatches to todos handler stub
- `gsd health` dispatches to health handler stub
- `gsd settings` dispatches to settings handler stub
- `gsd help` dispatches to help handler showing all available commands
- `gsd` with no subcommand shows same output as `gsd help`
- `gsd unknown-command` prints error listing available commands to stderr, exits 1
- All stub handlers respect the output mode (`--json`, `--plain`, rich default)
- All stubs return structured data (object with at minimum a `message` field) so formatOutput works

## Verification Criteria

- `node get-shit-done/bin/gsd-cli.cjs progress` from project root returns stub output without error
- `node get-shit-done/bin/gsd-cli.cjs todos` returns stub output
- `node get-shit-done/bin/gsd-cli.cjs health` returns stub output
- `node get-shit-done/bin/gsd-cli.cjs settings` returns stub output
- `node get-shit-done/bin/gsd-cli.cjs help` lists all available commands
- `node get-shit-done/bin/gsd-cli.cjs` (no args) shows help output
- `node get-shit-done/bin/gsd-cli.cjs nonsense` prints error to stderr listing available commands
- `node get-shit-done/bin/gsd-cli.cjs progress --json` returns valid JSON
- `node get-shit-done/bin/gsd-cli.cjs progress --plain` returns ANSI-free text
- `npm test` passes

<title>Add command routing and stub handlers to cli.cjs</title>
<description>
Update `get-shit-done/bin/lib/cli.cjs` to add:

**Command registry object `COMMANDS`:**
```javascript
const COMMANDS = {
  progress: { description: 'Show milestone progress and status', handler: handleProgress },
  todos: { description: 'List and inspect pending todos', handler: handleTodos },
  health: { description: 'Validate .planning/ directory integrity', handler: handleHealth },
  settings: { description: 'View and update configuration', handler: handleSettings },
  help: { description: 'Show available commands and usage', handler: handleHelp },
};
```

**Stub handler functions:**
Each stub handler receives `(projectRoot, args, mode)` and returns a result object.

- `handleProgress(projectRoot, args, mode)`: Return `{ command: 'progress', message: 'Progress command not yet implemented. See Phase 15.' }`
- `handleTodos(projectRoot, args, mode)`: Return `{ command: 'todos', message: 'Todos command not yet implemented. See Phase 16.' }`
- `handleHealth(projectRoot, args, mode)`: Return `{ command: 'health', message: 'Health command not yet implemented. See Phase 17.' }`
- `handleSettings(projectRoot, args, mode)`: Return `{ command: 'settings', message: 'Settings command not yet implemented. See Phase 18.' }`
- `handleHelp(projectRoot, args, mode)`: Build and return a help text listing all commands from the COMMANDS registry. Format:
  ```
  gsd - GSD CLI Utilities

  Usage: gsd <command> [options]

  Commands:
    progress    Show milestone progress and status
    todos       List and inspect pending todos
    health      Validate .planning/ directory integrity
    settings    View and update configuration
    help        Show available commands and usage

  Options:
    --json      Output as JSON
    --plain     Output as plain text (no colors)
  ```
  Return `{ command: 'help', message: helpText }` for JSON mode. For plain/rich mode, the text is the same (help has no ANSI).

**Router function `routeCommand(command, projectRoot, args, mode)`:**
- Look up `command` in `COMMANDS`
- If found: call the handler, return result
- If not found: return `null` (caller handles error)

Export `COMMANDS` and `routeCommand` in addition to existing exports.

<task id="2">
<title>Update CLI entry point with routing logic</title>
<description>
Update `get-shit-done/bin/gsd-cli.cjs` to use the command router:

**Updated main flow:**
1. Parse args
2. If no command or command is 'help': run help handler (does NOT require project root)
3. Find project root
4. If no project root: `cliError(...)` (unchanged from Plan 01)
5. Call `routeCommand(command, projectRoot, args, mode)`
6. If route returns null (unknown command): write error to stderr listing available commands from `COMMANDS` registry, exit 1
7. If route returns result: call `formatOutput(result, mode)` and write to stdout

**Unknown command error format:**
```
Error: Unknown command 'xyz'.

Available commands: progress, todos, health, settings, help
```

**Output handling:**
- For 'json' mode: `formatOutput` returns JSON string, write to stdout
- For 'plain' mode: extract `result.message` (string), strip ANSI, write to stdout
- For 'rich' mode: extract `result.message`, write to stdout as-is

This replaces the placeholder dispatch from Plan 01.
</description>
</task>
