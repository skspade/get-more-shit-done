The implementation per the design doc and CONTEXT.md locked decisions:

```bash
format_json_output() {
  local input
  input=$(cat)
  if echo "$input" | jq . >/dev/null 2>&1; then
    echo "$input" | jq .
  else
    echo "$input"
  fi
}
```

Key implementation details per CONTEXT.md locked decisions:
- Uses `cat` to buffer stdin (not line-by-line read) -- JSON payloads are multi-line
- Uses `jq .` for validation AND pretty-printing (2-space indent, jq default)
- No `--color-output` flag on jq (output often piped to tee/files)
- Function always exits 0 -- jq failure triggers raw echo fallback, never a non-zero exit
- No pipefail configuration inside the function -- script-wide `set -uo pipefail` (line 11) handles exit code propagation

Do NOT modify any other part of autopilot.sh. Insert only the function definition with a one-line comment above it.
