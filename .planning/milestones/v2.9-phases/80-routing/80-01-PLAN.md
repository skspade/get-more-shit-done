# Plan 80-01: Add routing logic to test-review command

## Objective

Replace the Step 12 placeholder in `test-review.md` with full routing logic: parse report summary for zero-recommendation detection, prompt user choice (quick task / milestone / done), execute the chosen route.

## Context

- `test-review.md` is a direct command file (no separate workflow). Steps 1-11 handle arg parsing, init, diff, agent spawn, report write, commit, and --report-only exit. Step 12 is a placeholder showing a completion banner.
- The routing pattern follows pr-review.md (quick route) and brainstorm.md (milestone delegation via SlashCommand).
- Unlike pr-review, test-review uses explicit user choice (no auto-scoring).
- Quick route groups findings by category (not file proximity).
- Init function already provides all needed data: `next_num`, `quick_dir`, `planner_model`, `executor_model`, `date`, `roadmap_exists`.

### 1b. Replace Step 12

Replace the entire `## 12. Display Completion` section with the following steps:

#### Step 12a: Parse Report Summary

After the agent returns (the structured return from `## REVIEWER COMPLETE`), parse the Summary table from the agent's output. Extract numeric values for:
- `Missing test files`
- `Coverage gaps`
- `Stale tests`
- `Consolidation opportunities`

Store these as `MISSING_COUNT`, `GAPS_COUNT`, `STALE_COUNT`, `CONSOLIDATION_COUNT`. Also calculate `TOTAL_FINDINGS = MISSING_COUNT + GAPS_COUNT + STALE_COUNT + CONSOLIDATION_COUNT`.

#### Step 12b: Zero-Recommendation Check

If `TOTAL_FINDINGS == 0`, display:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 GSD ► TEST REVIEW COMPLETE ✓
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

No issues found. All changed code has adequate test coverage.

Report: .planning/reviews/{date}-test-review.md
```

Exit. Do NOT proceed to routing.

#### Step 12c: User Choice Prompt

If findings exist, display the report content to the user, then prompt:

```
AskUserQuestion(
  header: "Test Review Routing",
  question: "How would you like to handle these findings?\n\n" +
    "Missing test files: {MISSING_COUNT}\n" +
    "Coverage gaps: {GAPS_COUNT}\n" +
    "Stale tests: {STALE_COUNT}\n" +
    "Consolidation opportunities: {CONSOLIDATION_COUNT}\n\n" +
    "Choose a route:",
  options: [
    "Quick task — create a quick task to address findings now",
    "Milestone — create a new milestone for comprehensive test improvement",
    "Done — keep the report, no further action"
  ]
)
```

Store the user's choice.

**If "Done":** Display completion banner and exit:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

```

Exit.


**Spawn planner (quick mode):**
