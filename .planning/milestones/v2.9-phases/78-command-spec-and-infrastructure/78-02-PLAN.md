Create `.claude/commands/gsd/test-review.md` following the direct agent spawn pattern from `audit-tests.md`.

**Frontmatter:**
```yaml
name: gsd:test-review
description: Analyze git diff for test coverage gaps, stale tests, and consolidation opportunities
argument-hint: "[--report-only]"
allowed-tools:
  - Read
  - Write
  - Edit
  - Glob
  - Grep
  - Bash
  - Task
  - AskUserQuestion
```

**Objective section:** Explain that this command gathers the git diff vs main, displays a banner with stats, spawns the gsd-test-reviewer agent for analysis, writes the report to disk, and optionally routes to quick task or milestone.

**Process steps (in order):**

**Step 1: Parse Arguments**
Parse `$ARGUMENTS` for `--report-only` flag. Store as `$REPORT_ONLY` boolean.

**Step 2: Initialize**
```bash
INIT=$(node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" init test-review)
```
Extract `reviewer_model`, `date`, `commit_docs` from JSON.

**Step 3: Resolve Diff Base**
Try `origin/main` first. Fallback chain: `origin/main` -> `origin/master` -> `main` -> `master`. Run `git fetch origin main --quiet 2>/dev/null || true` before resolving.

```bash
git fetch origin main --quiet 2>/dev/null || true
DIFF_BASE=""
for ref in origin/main origin/master main master; do
  if git rev-parse --verify "$ref" >/dev/null 2>&1; then
    DIFF_BASE="$ref"
    break
  fi
done
```

If no base found, display error and exit.

**Step 4: Gather Diff**
```bash
DIFF=$(git diff "$DIFF_BASE"...HEAD)
```

**If diff is empty:** Display "No changes found vs ${DIFF_BASE}. Nothing to review." and exit gracefully.

**Step 5: Measure Diff Size and Extract Files**
```bash
DIFF_LINES=$(echo "$DIFF" | wc -l)
CHANGED_FILES=$(echo "$DIFF" | grep '^diff --git' | sed 's|.*b/||')
FILE_COUNT=$(echo "$CHANGED_FILES" | wc -l)
```

**If DIFF_LINES > 2000 (large diff mode):**
- Get stat summary: `DIFF_STAT=$(git diff "$DIFF_BASE"...HEAD --stat)`
- Set `LARGE_DIFF=true`
- The agent input will use stat + file list instead of full diff

**Step 6: Gather Test Data**
```bash
TEST_COUNT=$(node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" test-count --raw 2>/dev/null || echo "0")
TEST_CONFIG=$(node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" test-config 2>/dev/null || echo "{}")
```

Parse budget from TEST_CONFIG: `per_phase` and `project` budget values. Calculate budget percentage.

**Step 7: Display Banner**
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 GSD ► TEST REVIEW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Changed files: {FILE_COUNT}
Test count: {TEST_COUNT}/{project_budget} ({percentage}%)
Diff size: {DIFF_LINES} lines {(summarized) if large}

◆ Spawning test reviewer...
```

**Step 8: Spawn Agent**
```
Task(
  prompt="First, read /Users/seanspade/.claude/agents/gsd-test-reviewer.md for your role and instructions.

<files_to_read>
- ./CLAUDE.md (if exists -- project instructions)
</files_to_read>

<test-review-input>
**Mode:** {standard | large-diff}
**Diff base:** {DIFF_BASE}
**Changed files ({FILE_COUNT}):**
{CHANGED_FILES list}

**Test count:** {TEST_COUNT}
**Test config:** {TEST_CONFIG JSON}

{If standard mode:}
**Full diff:**
```
{DIFF content}
```

{If large-diff mode:}
**Diff stat:**
```
{DIFF_STAT}
```
NOTE: Full diff exceeds 2000 lines. Use Read and Grep tools to inspect specific files as needed.
</test-review-input>

Analyze the diff and produce a structured test review report.",
  subagent_type="gsd-test-reviewer",
  model="{reviewer_model}",
  description="Test review analysis"
)
```

**Step 9: Write Report**
```bash
mkdir -p .planning/reviews
```
Write the agent's output to `.planning/reviews/{date}-test-review.md`.

**Step 10: Commit Report**
```bash
node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" commit "docs: test review report" --files ".planning/reviews/{date}-test-review.md"
```

**Step 11: Handle --report-only Exit**
**If `$REPORT_ONLY` is true:** Display "Report written to .planning/reviews/{date}-test-review.md" and exit. Do NOT proceed to routing.

**Step 12: Route (placeholder for Phase 80)**
Display the report to the user. Note that routing will be added in Phase 80.

```
 GSD ► TEST REVIEW COMPLETE ✓
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Report: .planning/reviews/{date}-test-review.md
```
