### Integration Issues (3 findings)

**INT-01 (P1): Missing phase complete after gate approval**
- File: `get-shit-done/scripts/autopilot.sh` lines 912-924
- After `run_verification_gate` returns 0 (human approved), autopilot.sh does not call `gsd_tools phase complete`. This creates an infinite verify loop on the gap path. The circuit breaker catches it after 3 no-progress iterations, but the expected behavior (advance to next phase) never occurs.
- Affected: VRFY-01

**INT-02 (P2): fix_desc not passed to agents**
- File: `get-shit-done/scripts/autopilot.sh` lines 764-776
- `run_fix_cycle` reads the human's description of what to fix via `read -r -p` but does not interpolate `$fix_desc` into the `/gsd:plan-phase --gaps` prompt. The description is echoed to console then discarded.
- Affected: VRFY-03

**INT-03 (P3): Source/installed drift**

- `autopilot.md` workflow file not tracked in source repo
- Runtime behavior correct; source verification would fail

## E2E Flow Verification (4/7 flows)

| # | Flow | Status | Notes |
|---|------|--------|-------|
| 1 | Cold start -> find first incomplete -> begin | PASS | find_first_incomplete_phase queries each phase |
| 2 | Resume -> --from-phase N -> skip to phase N | PASS | Direct CURRENT_PHASE assignment |
| 3 | Happy path inner chain: discuss --auto -> plan -> execute -> complete | PASS | Inner chain via Task sub-agents completes full phase |
| 4 | Execute failure -> debug-retry -> exhaust retries -> halt report | PASS | run_step_with_retry -> construct_debug_prompt -> write_failure_report |
| 5 | Gap path: execute gaps -> verify -> human gate -> next phase | BROKEN | Missing phase complete after gate approval (INT-01) |
| 6 | Fix cycle: human fix -> gap-closure -> re-verify | DEGRADED | fix_desc not passed to agents (INT-02) |
| 7 | Happy path gate: execute -> verify -> human gate | BYPASSED | execute-phase marks complete via inner chain; outer loop skips verify (architectural) |

### Architectural Note on Flow 7

The outer autopilot loop is a **fallback mechanism**. The `discuss` step spawns an inner chain (discuss -> plan -> execute via Task sub-agents) that completes an entire phase in one outer loop iteration. When execute-phase's internal verify passes, it calls `phase complete` directly. The outer loop's `plan`, `execute`, and `verify` steps only activate when the inner chain breaks at an intermediate step. The human verification gate (VRFY-01) is reached on the gap path only. This is architectural, not a wiring bug.
