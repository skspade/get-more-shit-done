# Milestone Context

**Source:** Brainstorm session (Automated UAT Session)
**Design:** .planning/designs/2026-03-22-automated-uat-session-design.md

## Milestone Goal

Implement automated UAT using Chrome MCP (primary) and Playwright (fallback) so the autopilot can automatically verify user-facing web UI behavior at the milestone level before completion. A Claude subagent acts as the tester — navigating the app, performing test scenarios from UAT.md files, taking screenshots as evidence, and reporting pass/fail. Failures feed into the existing gap closure loop for auto-fix cycles.

## Features

### UAT Test Source & Format

The automated UAT session reads from existing UAT.md files generated by the `verify-work` workflow. Each test has a name, expected behavior description, and result field. No changes to the UAT.md format are needed — the Claude agent interprets the `expected:` field as natural language instructions. A `uat-config.yaml` file in `.planning/` provides the app's base URL and optional startup command. The agent scans all phase directories for `*-UAT.md` files with `status: complete`. If no UAT.md files exist, the agent generates test scenarios from SUMMARY.md files.

### Chrome MCP Execution Engine

The core execution engine is a Claude subagent spawned with Chrome MCP access. Per test: navigate to the relevant page, interact with elements, take screenshots, use multimodal understanding to judge pass/fail against expected behavior, and record results with evidence. Screenshots saved to `.planning/uat-evidence/`.

### Playwright Fallback Engine

When Chrome MCP is unavailable (headless CI, no browser session), fall back to Playwright. Detection via attempting `tabs_context_mcp`. The agent generates ephemeral inline Playwright scripts per test, runs them, captures screenshots, and still uses Claude's judgment for pass/fail interpretation. No persistent test files generated.

### Autopilot Integration

New `runAutomatedUAT()` function in `autopilot.mjs` triggered after milestone audit passes, before milestone completion. Checks for `uat-config.yaml` (skips if missing). Spawns Claude session with `/gsd:uat-auto`. Failures feed into gap closure loop as `MILESTONE-UAT.md` gaps.

### UAT-Auto Workflow

New `/gsd:uat-auto` workflow: load config, discover tests, detect browser, start app if needed, execute tests sequentially, write results to `MILESTONE-UAT.md`, commit results, exit with status. Fully autonomous — no user interaction.

### Evidence & Reporting

Screenshots per test in `.planning/uat-evidence/{milestone}/`. `MILESTONE-UAT.md` with frontmatter status, results table, and YAML gaps section for failures. Evidence committed to git and cleaned up during milestone archival.
