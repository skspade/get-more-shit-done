# Plan 49-01: Debug Retry and Output Capture

## Objective

Add output-capturing step execution, debug retry loops, and failure reporting to `autopilot.mjs` — porting the behavior of bash `run_step_captured`, `run_step_with_retry`, `run_verify_with_debug_retry`, and their supporting functions.

## must_haves

- `runStepCaptured` captures stdout to a temp file while displaying to terminal
- `runStepWithRetry` retries a failing step up to `MAX_DEBUG_RETRIES` times, spawning a fresh debugger between retries
- `runVerifyWithDebugRetry` handles three cases: verify crash, gaps_found status, and success
- `constructDebugPrompt` builds the same XML-structured debug prompt as the bash version
- `writeFailureState` writes a blocker to STATE.md on retry exhaustion
- `clearFailureState` removes phase failure blockers from STATE.md on success
- `writeFailureReport` writes a `{padded}-FAILURE.md` file with failure details and resume instructions
- `MAX_DEBUG_RETRIES` reads from config via `getConfig('autopilot.max_debug_retries', 3)`
- On gaps_found with retries exhausted, `runVerifyWithDebugRetry` returns 0 so verification gate still presents

<task id="2">
<title>Implement runStepCaptured</title>
<actions>
1. Add function `runStepCaptured(prompt, stepName, outputFile)` that:
   - Calls `printBanner(`Phase ${CURRENT_PHASE} > ${stepName}`)`
   - Takes a progress snapshot before execution
   - In dry-run mode: writes `[DRY RUN] Would execute: ...` to both console and outputFile, checks progress, returns 0
   - For real execution: runs `claude -p --dangerously-skip-permissions --output-format json` via zx `$` with `.nothrow()`
   - Captures stdout: write result.stdout to the outputFile AND to process.stdout (simulating tee behavior)
   - Logs step start/done with exit code
   - Takes progress snapshot after, calls `checkProgress`
   - Returns the exit code (does NOT halt on failure — that's the caller's job)
2. Place this function right after the existing `runStep` function
</actions>
</task>

<task id="3">
<title>Implement constructDebugPrompt</title>
<actions>
1. Add function `constructDebugPrompt(stepName, exitCode, errorContext, phase, phaseDir, retryNum)` that returns a string with:
   - Slug: `autopilot-${stepName}-retry-${retryNum}`
   - Phase files: list CONTEXT.md, PLAN.md, and SUMMARY.md files from `PROJECT_DIR/${phaseDir}` using `fs.readdirSync` with filtering (not shell-out), prefixed with `- `
   - Template string containing XML blocks: `<objective>`, `<symptoms>`, `<mode>`, `<debug_file>`, `<files_to_read>` — matching the bash `construct_debug_prompt` output structure exactly
   - The `<symptoms>` block indents errorContext lines with 2 spaces
   - `<files_to_read>` includes the phase files plus `- .planning/STATE.md`
2. Place this function before `runStepCaptured`
</actions>
</task>

<task id="4">
<title>Implement writeFailureState, clearFailureState, writeFailureReport</title>
<actions>
1. Add `writeFailureState(stepName, exitCode, retryCount, maxRetries, outputFile)`:
   - Determine failureType: `"gaps_found"` if stepName is `"verify"` or `"verify-gaps"`, else `"exit_code"`
   - Collect debug session paths: `fs.readdirSync` on `.planning/debug/` filtering for `autopilot-*.md`, joined with `, ` (or `"None created"`)
   - Build blocker text: `[Phase ${CURRENT_PHASE} FAILURE]: type=${failureType} | step=${stepName} | retries=${retryCount}/${maxRetries} | exit_code=${exitCode} | debug_sessions=${debugSessions}`
   - Call `await gsdTools('state', 'add-blocker', '--text', blockerText)` wrapped in try/catch

2. Add `clearFailureState()`:
   - Call `await gsdTools('state', 'get', 'blockers')` to get existing blockers
   - If output contains `Phase ${CURRENT_PHASE} FAILURE`, extract the first matching line (trim leading `- `)
   - Call `await gsdTools('state', 'resolve-blocker', '--text', blockerText)` wrapped in try/catch

3. Add `writeFailureReport(stepName, exitCode, retryCount, maxRetries, outputFile)`:
   - Get phaseDir via `findPhaseInternal(PROJECT_DIR, CURRENT_PHASE)`
   - Compute paddedPhase: `String(CURRENT_PHASE).padStart(2, '0')`
   - Determine failureType same as writeFailureState
   - Collect debug sessions list with `- ` prefix per line
   - Read last 50 lines of outputFile via `fs.readFileSync` split by newline then slice(-50)
   - Write failure markdown to `${PROJECT_DIR}/${phaseDir.dir}/${paddedPhase}-FAILURE.md` with: Generated timestamp, Failure Type, Step, Exit Code, Retries, What Failed section, Failure Type description, Last Error Output in code block, Debug Sessions list, Resume instructions referencing `autopilot.sh --from-phase`
   - Log to stderr: `Failure report written to: ${failureFile}`
</actions>
</task>

<task id="5">
<title>Implement runStepWithRetry</title>
<actions>
1. Add `async function runStepWithRetry(prompt, stepName)`:
   - Initialize `retryCount = 0`
   - While loop:
     a. Create temp file via `os.tmpdir()` + `/gsd-autopilot-${Date.now()}` and push to `tempFiles` array
     b. Call `runStepCaptured(prompt, stepName, outputFile)` and capture exit code
     c. If exit 0: call `await clearFailureState()`, return 0
     d. Increment retryCount
     e. Log: `RETRY: step=${stepName} attempt=${retryCount}/${MAX_DEBUG_RETRIES} exit_code=${stepExit}`
     f. If retryCount > MAX_DEBUG_RETRIES: log exhaustion, call `await writeFailureState(...)` and `writeFailureReport(...)`, return stepExit
     g. Print retry banner: `◆ Debug retry ${retryCount}/${MAX_DEBUG_RETRIES} for step '${stepName}'...`
     h. Extract last 100 lines from outputFile as errorContext
     i. Get phaseDir from `findPhaseInternal(PROJECT_DIR, CURRENT_PHASE)`
     j. Call `constructDebugPrompt` to build debug prompt
     k. Print debug retry banner with box-drawing chars
     l. Spawn debugger: `await $\`cd ${PROJECT_DIR} && claude -p --dangerously-skip-permissions --output-format json ${debugPrompt}\`.nothrow()` — print stdout, do NOT halt on debugger failure (log warning instead)
     m. Loop continues to re-run the failed step
</actions>
</task>

<task id="6">
<title>Implement runVerifyWithDebugRetry</title>
<actions>
1. Add `async function runVerifyWithDebugRetry(phase)`:
   - Initialize `retryCount = 0`
   - While loop:
     a. Create temp file, push to tempFiles
     b. Call `runStepCaptured(\`/gsd:verify-work ${phase}\`, 'verify', outputFile)` capturing verifyExit

     c. **If verify crashed (verifyExit !== 0):**
        - Increment retryCount
        - If retryCount > MAX_DEBUG_RETRIES: write failure state/report for "verify", return verifyExit
        - Print retry message for verify crash
        - Extract error context (last 100 lines)
        - Get phaseDir, build debug prompt, print banner, spawn debugger (nothrow, ignore failure)
        - Continue loop

     d. **If verify succeeded (verifyExit === 0):**
        - Get phaseDir from `findPhaseInternal(PROJECT_DIR, phase)`
        - Call `getVerificationStatus(PROJECT_DIR, phaseDir.dir)` to get `{ status, score }`
        - If status is NOT `"gaps_found"`: call `await clearFailureState()`, return 0
        - **Gaps found path:**
          - Increment retryCount
          - If retryCount > MAX_DEBUG_RETRIES: write failure state/report for "verify-gaps" with exit code 0, return 0 (so gate still presents)
          - Log gaps retry
          - Call `getGapsSummary(PROJECT_DIR, phaseDir.dir)` for gaps text
          - Build error context: `"Verification status: gaps_found\nGaps:\n${gapsSummary}"`
          - Build debug prompt with stepName="verify-gaps", exitCode=0
          - Print gap fix banner
          - Spawn debugger (nothrow)
          - Continue loop
</actions>
</task>
