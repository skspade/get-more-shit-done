# Plan 49-02: Verification Gate

## Objective

Add TTY-based verification gate to `autopilot.mjs` with approve/fix/abort routing — porting the behavior of bash `run_verification_gate`, `run_fix_cycle`, `print_verification_gate`, `handle_abort`, and `extract_autonomous_decisions`.

## must_haves

- `printVerificationGate` displays checkpoint box with phase, status, score, gaps, and autonomous decisions
- `runVerificationGate` loops until valid input, re-presenting after fix cycles
- Input accepts aliases: `a`/`approve`/`yes`/`y`, `f`/`fix`, `x`/`abort`/`quit`/`q` (case-insensitive, trimmed)
- `runFixCycle` prompts for description via TTY, runs plan-phase --gaps, execute-phase --gaps-only, and verify-work
- `handleAbort` prints state-preserved message and exits with code 2
- `extractAutonomousDecisions` reads CONTEXT.md, checks for auto-context markers, extracts Claude's Decision lines
- In dry-run mode, verification gate is auto-approved
- Readline uses `/dev/tty` for input (not stdin, which may be piped)
- Fix cycle resets circuit breaker before and after

## Tasks

<task id="1">
<title>Implement extractAutonomousDecisions</title>
<actions>
1. Add function `extractAutonomousDecisions(phaseDir)`:
   - Use `fs.readdirSync` on `path.join(PROJECT_DIR, phaseDir)` to find files matching `*-CONTEXT.md`
   - If no context file found, return empty string
   - Read the context file content
   - Check if content contains `"auto-context"` or `"Auto-generated"` — if neither, return empty string
   - Filter lines containing `"(Claude's Decision:"`
   - For each matching line, trim leading whitespace and `- `, then re-prefix with `  - `
   - Return joined result string (or empty string if no matches)
</actions>
</task>

<task id="2">
<title>Implement printVerificationGate and handleAbort</title>
<actions>
1. Add function `printVerificationGate(phase, status, score, decisions, gaps)`:
   - Print empty line
   - Print checkpoint box: `\u2554` + `\u2550`.repeat(62) + `\u2557`, `\u2551  CHECKPOINT: Verification Required` padded to 62 + `\u2551`, `\u255a` + `\u2550`.repeat(62) + `\u255d`
   - Print empty line
   - Print `Phase ${phase} verification: ${status} (score: ${score})`
   - If status is `"gaps_found"` and gaps is non-empty: print `"Gaps found:"`, print gaps, print empty line
   - If decisions is non-empty: print `"Decisions Made Autonomously:"`, print decisions, print empty line
   - Print separator line: `\u2500`.repeat(62)
   - Print routing options: `\u2192 approve  \u2014 continue to next phase`, `\u2192 fix      \u2014 describe issues and trigger gap-closure cycle`, `\u2192 abort    \u2014 stop autopilot cleanly (state preserved)`
   - Print separator line

2. Add function `handleAbort(phase)`:
   - Print: `Autopilot aborted by user at Phase ${phase} verification.`
   - Print: `State preserved. To resume:`
   - Print: `  autopilot.sh --from-phase ${phase} --project-dir ${PROJECT_DIR}`
   - Call `process.exit(2)`
</actions>
</task>

<task id="3">
<title>Implement runFixCycle</title>
<actions>
1. Add `import readline from 'readline'` at the top of the file (or use `import { createInterface } from 'readline'`)
2. Add helper function `askTTY(prompt)` that:
   - Creates a readline interface with `input: fs.createReadStream('/dev/tty')` and `output: process.stdout`
   - Returns a promise that resolves with the trimmed answer
   - Closes the interface after getting the answer
3. Add `async function runFixCycle(phase)`:
   - Call `const fixDesc = await askTTY('Describe what to fix: ')`
   - Print: `Running gap-closure cycle for Phase ${phase}...`
   - Print: `Fix request: ${fixDesc}`
   - Reset circuit breaker: `noProgressCount = 0`
   - Run three steps using the existing `runStep` function (NOT runStepWithRetry — matching bash which uses plain run_step for fix cycle):
     a. `await runStep(\`/gsd:plan-phase ${phase} --gaps -- Human fix request: ${fixDesc}\`, 'fix-plan')`
     b. `await runStep(\`/gsd:execute-phase ${phase} --gaps-only -- Human fix request: ${fixDesc}\`, 'fix-execute')`
     c. `await runStep(\`/gsd:verify-work ${phase}\`, 'fix-verify')`
   - Reset circuit breaker after: `noProgressCount = 0`
</actions>
</task>

<task id="4">
<title>Implement runVerificationGate</title>
<actions>
1. Add `async function runVerificationGate(phase)`:
   - While loop:
     a. Get phaseDir from `findPhaseInternal(PROJECT_DIR, phase)` — use `phaseDir.dir` for the directory path
     b. Call `getVerificationStatus(PROJECT_DIR, phaseDir.dir)` to get `{ status, score }`
     c. Call `extractAutonomousDecisions(phaseDir.dir)` for decisions string
     d. If status is `"gaps_found"`: call `getGapsSummary(PROJECT_DIR, phaseDir.dir)` and join the result array into a string, else set gaps to empty string
     e. Call `printVerificationGate(phase, status, score, decisions, gaps)`
     f. Call `const response = await askTTY('\u2192 ')` and normalize: lowercase, trim
     g. Switch on response:
        - `'a'`, `'approve'`, `'yes'`, `'y'`: print `\u2713 Phase ${phase} approved. Continuing...`, return
        - `'f'`, `'fix'`: call `await runFixCycle(phase)`, continue loop (re-presents gate)
        - `'x'`, `'abort'`, `'quit'`, `'q'`: call `handleAbort(phase)`
        - default: print `Unknown response: '${response}'`, print `Enter: approve / fix / abort`, continue loop
</actions>
</task>

<task id="5">
<title>Wire verification gate into main loop verify case</title>
<actions>
1. Replace the existing `verify` case in the main loop switch statement. The current code is:

   ```
   case 'verify':
     // Phase 48: simple runStep (Phase 49 adds runVerifyWithDebugRetry + verification gate)
     await runStep(`/gsd:verify-work ${CURRENT_PHASE}`, 'verify');

     // Uses gsdTools shell-out per CONTEXT.md decision

       console.log('[DRY RUN] Auto-approving verification gate');
     }
     await gsdTools('phase', 'complete', CURRENT_PHASE);
     break;
   ```
   Replace with:
   ```
   case 'verify':
     await runVerifyWithDebugRetry(CURRENT_PHASE);
     if (DRY_RUN) {
       console.log('[DRY RUN] Auto-approving verification gate');
     } else {
       await runVerificationGate(CURRENT_PHASE);
     }
     await gsdTools('phase', 'complete', CURRENT_PHASE);
     break;
   ```
   Note: `runVerifyWithDebugRetry` handles the verify exit code internally (returns 0 even on gaps_found exhaustion). The verification gate presents regardless.

2. Also handle the verify exit code for fatal failures. Wrap with:
   ```
   const verifyExit = await runVerifyWithDebugRetry(CURRENT_PHASE);
   if (verifyExit !== 0) {
     printHaltReport('Verification failed after debug retries', 'verify', verifyExit);
     process.exit(1);
   }
   ```
</actions>
