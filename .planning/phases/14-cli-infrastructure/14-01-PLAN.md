# Plan 01: CLI Entry Point and Core Infrastructure

## Objective

Create the `gsd` CLI binary entry point with project discovery (walk-up `.planning/` finder), argument parsing (output flags), output formatting utility, and error handling. After this plan, `gsd` can be invoked from any subdirectory, discovers the project root, parses `--json`/`--plain` flags, and shows errors when run outside a GSD project.

## Must-Haves (Goal-Backward)

- `gsd` binary is executable with `#!/usr/bin/env node` shebang
- Running from any subdirectory finds `.planning/` by walking up directory tree
- `--json` flag causes all output to be valid JSON on stdout
- `--plain` flag causes all output to be ANSI-free text on stdout
- Running outside a GSD project prints clear error to stderr with exit code 1
- Error messages name `.planning/` and suggest where to run

## Verification Criteria

- `node get-shit-done/bin/gsd-cli.cjs` executes without crash from project root
- `node get-shit-done/bin/gsd-cli.cjs` from a subdirectory discovers `.planning/`
- `node get-shit-done/bin/gsd-cli.cjs --json` outputs valid JSON
- `node get-shit-done/bin/gsd-cli.cjs --plain` outputs no ANSI escape codes
- Running from `/tmp` prints error to stderr and exits with code 1
- `npm test` passes with new test file

<task id="1">
<title>Create output formatting module</title>
<description>
Create `get-shit-done/bin/lib/cli.cjs` with the output formatting and project discovery utilities.

**Project discovery function `findProjectRoot(startDir)`:**
- Walk up from `startDir` (default: `process.cwd()`) looking for a `.planning/` directory
- Stop at filesystem root
- Return the directory containing `.planning/`, or `null` if not found

**Output formatter `formatOutput(data, mode)`:**
- `mode` is one of: `'json'`, `'plain'`, `'rich'` (default)
- `'json'` mode: `JSON.stringify(data, null, 2)`
- `'plain'` mode: strip ANSI codes from the string representation. Use a regex to strip `\x1b\[[0-9;]*m` sequences.
- `'rich'` mode: pass through as-is (ANSI colors allowed)
- Return the formatted string

**Argument parsing function `parseArgs(argv)`:**
- Accept `process.argv.slice(2)`
- Extract `--json` flag -> mode = 'json'
- Extract `--plain` flag -> mode = 'plain'
- Default mode = 'rich'
- Extract subcommand (first non-flag argument)
- Extract remaining positional args
- Return `{ command, args, mode }`

**Error helper `cliError(message)`:**
- Write `Error: {message}` to stderr
- Exit with code 1

Export all four functions.

Follow existing CJS pattern: `require()` / `module.exports`, no ESM. No external dependencies.
</description>
</task>

<task id="2">
<title>Create CLI entry point binary</title>
<description>
Create `get-shit-done/bin/gsd-cli.cjs` as the executable entry point.

**File structure:**
```javascript
#!/usr/bin/env node
const { findProjectRoot, parseArgs, formatOutput, cliError } = require('./lib/cli.cjs');
```

**Main flow:**
1. Parse args with `parseArgs(process.argv.slice(2))`
2. Call `findProjectRoot(process.cwd())`
3. If no project root found AND command is not 'help':
   - Call `cliError('Not a GSD project (no .planning/ directory found).\nRun this command from within a GSD project directory.')
4. If project root found, store it for handler dispatch
5. For now, if a command is given, print a placeholder message indicating the command was received (this will be replaced in Plan 02)
6. If no command given, print usage summary (same as help stub)

**Make executable:** File must have `chmod +x` applied. The shebang line handles this on Unix.

Keep the main flow simple -- this is a thin dispatcher. Plan 02 adds the actual command routing.
</description>
</task>
