**What:** Replace the placeholder "Steps 4-11" comment in `get-shit-done/workflows/pr-review.md` with Step 4 that deduplicates and groups the `$FINDINGS` array by file proximity.

**Implementation:**

Replace the block starting at `**Steps 4-11: Deduplication, Persistence, Scoring, and Routing**` with Step 4 content. Keep a placeholder for Steps 7-11 (Phases 42-43).

Step 4 instructions for the workflow (written as orchestrator instructions, not code):

1. **Sort** `$FINDINGS` by `file` path (alphabetical), then by `line` number (ascending). Findings with null file/line go last.

2. **Group by proximity:** Iterate sorted findings. For each finding, if it shares the same `file` as the previous finding AND the `line` difference is <= 20, add it to the current group. Otherwise, start a new group. Findings with null file/line each become their own single-finding group.

3. **Merge overlapping groups transitively:** After initial grouping, check if any adjacent groups share the same file and have overlapping or touching line ranges (group A's end line + 20 >= group B's start line). If so, merge them. Repeat until no more merges occur.

4. **Assign group metadata:** For each group:
   - `id`: `"group-N"` (sequential from 1)
   - `primary_file`: the shared file path
   - `line_range`: `[min_line, max_line]` across all findings in the group
   - `findings`: array of original finding objects
   - `max_severity`: highest severity using priority order critical > important > suggestion
   - `agents_involved`: deduplicated list of agent names from findings

5. **Store** result as `$GROUPS` array.

6. **Display dedup summary** using this exact format:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 GSD ► PR REVIEW FINDINGS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Raw findings: {N}
After dedup: {N} file-region groups

◆ {group.primary_file}:{group.line_range[0]}-{group.line_range[1]} — {group.max_severity} ({N} findings from {agents_involved joined with ", "})
◆ ...
```

For single-line groups (where line_range start == end), display as `{file}:{line}` without the range dash.
