  <action>
Add two new exported functions to phase.cjs:

**`findFirstIncompletePhase(cwd)`:**
1. Call `roadmap.cmdRoadmapAnalyze(cwd, true)` to get the phase list — but since `cmdRoadmapAnalyze` uses `output()` which writes to stdout, we need a different approach. Instead, require `roadmap.cjs` at the top of the file and use an internal approach:
   - Read ROADMAP.md directly (same as cmdRoadmapAnalyze does)
   - Or capture the output by calling `cmdRoadmapAnalyze` with `raw=true` — but that still outputs to stdout
   - Best approach: Extract the roadmap parsing into a shared internal, or simply replicate the phase iteration pattern inline
   - Actually, the simplest approach per CONTEXT.md: iterate phases from roadmap analysis. To avoid stdout issues, use `findPhaseInternal` + `computePhaseStatus` for each phase extracted from the roadmap

   Concrete implementation:
   - Read ROADMAP.md content from `path.join(cwd, '.planning', 'ROADMAP.md')`
   - Use the same regex pattern as `cmdRoadmapAnalyze` to extract phase numbers: `/#{2,4}\s*Phase\s+(\d+[A-Z]?(?:\.\d+)*)\s*:\s*([^\n]+)/gi`
   - Sort phases using `comparePhaseNum`
   - For each phase in order, call `findPhaseInternal(cwd, phaseNum)` to get phaseInfo
   - If phaseInfo exists, call `computePhaseStatus(cwd, phaseInfo)` and check `phase_complete`
   - If phaseInfo does not exist (no directory), the phase is incomplete — return it
   - Return the first phase number (as string) where `phase_complete === false` (or no directory exists)
   - Return `null` if all phases are complete

**`nextIncompletePhase(cwd, currentPhase)`:**
1. Same approach as `findFirstIncompletePhase` but:
   - Convert `currentPhase` to a comparable form using `normalizePhaseName`
   - Skip all phases where `comparePhaseNum(phaseNum, currentPhase) <= 0`
   - Return the first incomplete phase number after `currentPhase`
   - Return `null` if no incomplete phases follow

Add both functions to the `module.exports` block at the bottom of the file.

Add `const roadmapPath = ...` as needed — do NOT add a `require('./roadmap.cjs')` to avoid circular dependency concerns. The functions should read ROADMAP.md directly using `fs` and `path`.
