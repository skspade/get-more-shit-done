  <action>
Add two new exported functions to verify.cjs, placed before the `module.exports` block:

**`getVerificationStatus(cwd, phaseDir)`:**
1. `phaseDir` is the relative phase directory path (e.g., `.planning/phases/47-cjs-module-extensions`)
2. Build the absolute path: `path.join(cwd, phaseDir)`
3. List files in that directory
4. Search order: find `*-VERIFICATION.md` first, then `*-UAT.md` fallback
5. If found, read file content and call `extractFrontmatter(content)`
6. Return `{ status: fm.status || null, score: fm.score || null }` from parsed frontmatter
7. Return `null` if neither file exists or directory cannot be read

**`getGapsSummary(cwd, phaseDir)`:**
1. Same file search as `getVerificationStatus` — find VERIFICATION.md or UAT.md
2. If found, extract lines from `## ...Gap` sections using string operations:
   - Find all sections whose heading matches `/^## .*[Gg]ap/m`
   - For each matching section, collect non-empty, non-heading lines until the next `## ` heading
   - These are the gap description lines
3. Return an array of trimmed, non-empty gap description strings
4. Return empty array `[]` if no verification file exists or no gap sections found

Both functions use `fs` and `path` (already imported in verify.cjs). Both use `extractFrontmatter` (already imported).

Add both to the `module.exports` block.
