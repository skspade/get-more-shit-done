<task type="auto">
  <name>Task 1: Implement cmdPhaseStatus in phase.cjs</name>
  <files>.claude/get-shit-done/bin/lib/phase.cjs</files>
  <action>
Add a new `cmdPhaseStatus(cwd, phaseArg, raw)` function to phase.cjs. This function:

1. Uses `findPhaseInternal(cwd, phaseArg)` to locate the phase directory and get artifact inventory
2. Reads the phase directory to detect artifact presence:
   - `has_context`: any file matching `*-CONTEXT.md`
   - `has_research`: any file matching `*-RESEARCH.md`
   - `has_plans`: any file matching `*-PLAN.md`
   - `plan_count`: count of `*-PLAN.md` files
   - `has_summaries`: any file matching `*-SUMMARY.md`
   - `summary_count`: count of `*-SUMMARY.md` files
   - `all_plans_have_summaries`: plan_count > 0 AND summary_count >= plan_count
   - `has_verification`: any file matching `*-VERIFICATION.md`
3. Infers the current lifecycle step from artifact state:
   - `"discuss"` if !has_context
   - `"plan"` if has_context AND !has_plans
   - `"execute"` if has_plans AND !all_plans_have_summaries
   - `"verify"` if all_plans_have_summaries AND !has_verification
   - `"complete"` if has_verification
4. Returns JSON via `output()`:
```json
{
  "phase": "1",
  "phase_dir": ".planning/phases/01-core-loop-infrastructure",
  "step": "discuss|plan|execute|verify|complete",
  "has_context": true/false,
  "has_research": true/false,
  "has_plans": true/false,
  "plan_count": N,
  "has_summaries": true/false,
  "summary_count": N,
  "all_plans_have_summaries": true/false,
  "has_verification": true/false,
  "phase_complete": true/false
}
```

Export `cmdPhaseStatus` in the module.exports at the bottom of the file.

Do NOT use the `extractFrontmatter` function for artifact detection -- just use filename glob matching via `fs.readdirSync`. Keep it simple: file presence is the signal, not file contents.
