**Header and argument parsing:**
```bash
#!/usr/bin/env bash
set -euo pipefail
```

Parse arguments:
- `--from-phase N`: Start/resume from phase N (optional, defaults to detecting from state)
- `--project-dir PATH`: Project directory (optional, defaults to current directory)
- `--dry-run`: Print what would be done without executing (optional)
- First positional arg or no args: auto-detect current milestone

**Helper functions:**

1. `gsd_tools()` - Wrapper that calls `node "$GSD_HOME/bin/gsd-tools.cjs" "$@" --cwd "$PROJECT_DIR"` where GSD_HOME resolves to the directory containing the get-shit-done framework (use `SCRIPT_DIR` to find it relative to the script location: `$SCRIPT_DIR/..`).

2. `print_banner(text)` - Prints GSD-style separator banner:
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 GSD ► AUTOPILOT: $text
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

3. `get_phase_status(phase)` - Calls `gsd_tools phase-status "$phase"` and returns JSON.

4. `get_all_phases()` - Calls `gsd_tools roadmap analyze` and extracts phase list.

5. `get_config(key)` - Calls `gsd_tools config-get "$key"` with fallback default.

**Main initialization:**
- Resolve PROJECT_DIR (default: pwd)
- Resolve GSD_HOME from SCRIPT_DIR
- Validate jq is available (exit with error if not)
- Validate .planning/ exists (exit with error if not)
- Read circuit breaker threshold from config (default: 3)
- Determine starting phase: use --from-phase if provided, otherwise find first incomplete phase from roadmap analyze
- Print startup banner showing milestone name, phase range, starting phase

**Main loop:**
```bash
CURRENT_PHASE="$START_PHASE"
while true; do
  STATUS_JSON=$(get_phase_status "$CURRENT_PHASE")
  STEP=$(echo "$STATUS_JSON" | jq -r '.step')
  PHASE_DIR=$(echo "$STATUS_JSON" | jq -r '.phase_dir')

  case "$STEP" in
    discuss)  run_step "/gsd:discuss-phase $CURRENT_PHASE --auto" ;;
    plan)     run_step "/gsd:plan-phase $CURRENT_PHASE --auto" ;;
    execute)  run_step "/gsd:execute-phase $CURRENT_PHASE" ;;
    verify)   run_step "/gsd:verify-work $CURRENT_PHASE" ;;
    complete)
      print_banner "Phase $CURRENT_PHASE COMPLETE ✓"
      CURRENT_PHASE=$(next_incomplete_phase "$CURRENT_PHASE")
      if [ -z "$CURRENT_PHASE" ]; then
        print_banner "ALL PHASES COMPLETE ✓"
        print_final_report
        exit 0
      fi
      continue  # Skip circuit breaker check for phase transitions
      ;;
    *)
      echo "ERROR: Unknown step '$STEP' for phase $CURRENT_PHASE" >&2
      exit 1
      ;;
  esac
done
```

6. `run_step(prompt)` - Core execution function:
```bash
run_step() {
  local prompt="$1"
  local snapshot_before=$(take_progress_snapshot)

  print_banner "Phase $CURRENT_PHASE > $STEP"

  if [ "$DRY_RUN" = true ]; then
    echo "[DRY RUN] Would execute: claude -p --dangerously-skip-permissions \"$prompt\""
    return 0
  fi

  # Execute Claude Code with fresh context window
  local exit_code=0
  claude -p --dangerously-skip-permissions --output-format json "$prompt" || exit_code=$?

  local snapshot_after=$(take_progress_snapshot)
  check_progress "$snapshot_before" "$snapshot_after" "$STEP"

  if [ $exit_code -ne 0 ]; then
    # Check if artifacts changed despite non-zero exit (partial success)
    if [ "$snapshot_before" = "$snapshot_after" ]; then
      echo "ERROR: Step failed with exit code $exit_code and no artifacts created" >&2
      print_halt_report "Step failure" "$STEP" "$exit_code"
      exit 1
    else
      echo "WARNING: Step exited with code $exit_code but made progress. Continuing." >&2
    fi
  fi
}
```

7. `next_incomplete_phase(current)` - Gets next phase number from roadmap that isn't complete.

8. `take_progress_snapshot()` - Returns a string of progress signals:
   - git commit count: `git rev-list --count HEAD 2>/dev/null || echo 0`
   - artifact file count in current phase dir: count of *.md files
   - Use pipe-delimited format: `"commits|artifacts"`

9. `check_progress(before, after, step)` - Compares snapshots, increments/resets no-progress counter.

10. `print_halt_report(reason, step, exit_code)` - Structured halt output showing:
    - Which phase and step
    - Last N iteration descriptions from ITERATION_LOG array
    - What progress signals were checked
    - How to resume: `autopilot.sh --from-phase N`

11. `print_final_report()` - End-of-run summary showing phases completed.

**CRITICAL implementation notes:**
- NEVER use `claude -c` or `--resume`. Each `claude -p` call MUST be a fresh process (LOOP-06, LOOP-08).
- Always pass `--dangerously-skip-permissions` (LOOP-07).
- Always pass `--output-format json` for scriptability.
- The `claude` CLI must be on PATH. If not found, exit with helpful error.
- Use `set -euo pipefail` at top but handle claude exit codes explicitly (don't let set -e kill the script on claude non-zero).
- Track iterations in a bash array `ITERATION_LOG` for halt report.
