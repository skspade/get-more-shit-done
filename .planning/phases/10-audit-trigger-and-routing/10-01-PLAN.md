# Plan 10-01: Audit Trigger and Routing in Autopilot

## Objective

Replace autopilot's current "all phases complete -> exit 0" path with a milestone audit trigger that invokes `/gsd:audit-milestone`, parses the result status, and routes to the correct exit code based on outcome (passed, gaps_found, tech_debt) with configurable tech debt handling.

## must_haves

- [ ] When `next_incomplete_phase` returns empty, autopilot invokes `/gsd:audit-milestone` instead of calling `print_final_report` and exiting
- [ ] Audit result status is parsed from the generated `v*-MILESTONE-AUDIT.md` frontmatter using `gsd-tools frontmatter get`
- [ ] Three-way routing: `passed` -> exit 0, `gaps_found` -> exit 10, `tech_debt` -> conditional on config
- [ ] `auto_accept_tech_debt` config read via `get_config "autopilot.auto_accept_tech_debt" "true"`
- [ ] When `auto_accept_tech_debt` is true, `tech_debt` status is treated as passed (exit 0)
- [ ] When `auto_accept_tech_debt` is false, `tech_debt` status is treated as gaps_found (exit 10)
- [ ] Audit invocation uses `run_step_with_retry` for resilience
- [ ] All new code is reachable from the existing main `while true` loop (no dead code paths)

## Context

**Insertion point:** Lines 928-941 of `autopilot.sh` — the `complete)` case in the main loop where `next_incomplete_phase` returns empty and the script currently calls `print_final_report` then `exit 0`.

**Existing helpers to reuse:**
- `get_config` (line 97): Reads config.json with default fallback
- `run_step_with_retry` (line 398): Runs Claude step with debug-retry on failure
- `print_banner` (line 84): Consistent banner formatting
- `print_final_report` (line 187): Current "all done" message (will be wrapped/called before audit)

**Exit code contract:**
- Exit 0: audit passed (or tech_debt accepted) — signals Phase 12 (milestone completion)
- Exit 10: gaps found (or tech_debt rejected) — signals Phase 11 (gap closure loop)
- Existing codes (1, 2, 130) are NOT affected

**Audit file location:** `.planning/v*-MILESTONE-AUDIT.md` (found via `ls -t | head -1`)
**Audit frontmatter:** `status: passed | gaps_found | tech_debt` (YAML)

## Tasks

<task id="1" title="Add audit trigger and routing function">

### Description

Create a `run_milestone_audit` function in `autopilot.sh` that:
1. Prints a banner: `MILESTONE AUDIT`
2. Invokes `/gsd:audit-milestone` via `run_step_with_retry`
3. Locates the generated audit file via `ls -t .planning/v*-MILESTONE-AUDIT.md 2>/dev/null | head -1`
4. Parses the `status` field from frontmatter using `gsd_tools frontmatter get "$audit_file" --field status --raw`
5. Reads `auto_accept_tech_debt` config via `get_config "autopilot.auto_accept_tech_debt" "true"`
6. Routes based on status:
   - `passed`: prints success banner, returns 0
   - `gaps_found`: prints gaps banner, returns 10
   - `tech_debt` + config true: prints accepted banner, returns 0
   - `tech_debt` + config false: prints gaps banner, returns 10
   - Unknown/missing: prints error, returns 1

**Insert location:** After the `print_final_report` function (around line 196), before the "Step Execution" section. This keeps all utility functions together.

**Implementation pattern:**

```bash
# ─── Milestone Audit ─────────────────────────────────────────────────────────

run_milestone_audit() {
  print_banner "MILESTONE AUDIT"

  # Invoke audit-milestone via claude with retry support
  local audit_prompt="/gsd:audit-milestone"
  local audit_exit=0
  run_step_with_retry "$audit_prompt" "milestone-audit" || audit_exit=$?

  if [[ $audit_exit -ne 0 ]]; then
    echo "ERROR: Milestone audit failed (exit $audit_exit)" >&2
    return 1
  fi

  # Locate the most recent audit file
  local audit_file
  audit_file=$(ls -t "$PROJECT_DIR/.planning/v"*"-MILESTONE-AUDIT.md" 2>/dev/null | head -1)

  if [[ -z "$audit_file" ]]; then
    echo "ERROR: No MILESTONE-AUDIT.md file found after audit" >&2
    return 1
  fi

  # Parse audit status from frontmatter
  local audit_status
  audit_status=$(gsd_tools frontmatter get "$audit_file" --field status --raw 2>/dev/null || echo "")

  if [[ -z "$audit_status" ]]; then
    echo "ERROR: Could not parse status from $audit_file" >&2
    return 1
  fi

  echo "Audit result: $audit_status"

  # Read tech debt config
  local auto_accept_tech_debt
  auto_accept_tech_debt=$(get_config "autopilot.auto_accept_tech_debt" "true")

  # Route based on status
  case "$audit_status" in
    passed)
      print_banner "MILESTONE AUDIT PASSED ✓"
      echo "All requirements satisfied. Ready for milestone completion."
      return 0
      ;;
    gaps_found)
      print_banner "MILESTONE AUDIT: GAPS FOUND"
      echo "Audit found gaps that need fixing."
      return 10
      ;;
    tech_debt)
      if [[ "$auto_accept_tech_debt" == "true" ]]; then
        print_banner "MILESTONE AUDIT PASSED ✓ (tech debt accepted)"
        echo "Audit found tech debt only. auto_accept_tech_debt=true — treating as passed."
        return 0
      else
        print_banner "MILESTONE AUDIT: GAPS FOUND (tech debt rejected)"
        echo "Audit found tech debt. auto_accept_tech_debt=false — treating as gaps."
        return 10
      fi
      ;;
    *)
      echo "ERROR: Unknown audit status '$audit_status'" >&2
      return 1
      ;;
  esac
}
```

- [ ] `run_milestone_audit` function exists in autopilot.sh

- [ ] Function uses `gsd_tools frontmatter get` to parse status
- [ ] Function reads `autopilot.auto_accept_tech_debt` config with default `true`
- [ ] All three status values (passed, gaps_found, tech_debt) are handled
- [ ] Tech debt routing respects config value

<task id="2" title="Wire audit trigger into main loop" depends_on="1">

### Description

Modify the `complete)` case in the main loop to call `run_milestone_audit` when no more incomplete phases remain, instead of just calling `print_final_report` and exiting.

**Current code (lines 928-941):**

```bash
    complete)
      print_banner "Phase $CURRENT_PHASE COMPLETE ✓"

      NEXT_PHASE=$(next_incomplete_phase "$CURRENT_PHASE")
      if [[ -z "$NEXT_PHASE" ]]; then
        print_final_report
        exit 0
      fi

      CURRENT_PHASE="$NEXT_PHASE"
      # Reset no-progress counter on phase advancement (this IS progress)
      NO_PROGRESS_COUNT=0
      ITERATION_LOG=()
      continue
      ;;
```

**Replace with:**

```bash
    complete)
      print_banner "Phase $CURRENT_PHASE COMPLETE ✓"

      NEXT_PHASE=$(next_incomplete_phase "$CURRENT_PHASE")
      if [[ -z "$NEXT_PHASE" ]]; then
        print_final_report

        # All phases complete — trigger milestone audit
        local audit_result=0
        run_milestone_audit || audit_result=$?

        if [[ $audit_result -eq 0 ]]; then
          # Audit passed (or tech debt accepted) — signal milestone completion
          exit 0
        elif [[ $audit_result -eq 10 ]]; then
          # Gaps found (or tech debt rejected) — signal gap closure needed
          exit 10
        else
          # Audit failed with error
          echo "ERROR: Milestone audit encountered an error" >&2
          exit 1
        fi
      fi

      CURRENT_PHASE="$NEXT_PHASE"
      # Reset no-progress counter on phase advancement (this IS progress)
      NO_PROGRESS_COUNT=0
      ITERATION_LOG=()
      continue
      ;;
```

This preserves `print_final_report` (shows iteration stats) before the audit, and translates the audit function's return code to the script's exit code. The exit codes (0, 10, 1) become the contract for Phase 11 (gap closure) and Phase 12 (completion) to consume.

### Verification

- [ ] `complete)` case calls `run_milestone_audit` when `NEXT_PHASE` is empty
- [ ] `print_final_report` is still called before audit (preserves stats output)
- [ ] Exit 0 when audit returns 0 (passed/accepted tech debt)
- [ ] Exit 10 when audit returns 10 (gaps found/rejected tech debt)
- [ ] Exit 1 when audit returns other error codes
- [ ] Phase advancement path (non-empty NEXT_PHASE) is NOT affected
- [ ] No dead code paths — audit trigger is reachable from the main while loop

### Description

if [[ -n "$FROM_PHASE" ]]; then

else

**Replace with:**

```bash
if [[ -n "$FROM_PHASE" ]]; then
  CURRENT_PHASE="$FROM_PHASE"
else
  CURRENT_PHASE=$(find_first_incomplete_phase)
  if [[ -z "$CURRENT_PHASE" ]]; then
    print_banner "ALL PHASES COMPLETE ✓"
    echo "All phases in the milestone are already complete."

    # All phases already complete — trigger milestone audit
    local audit_result=0
    run_milestone_audit || audit_result=$?

    if [[ $audit_result -eq 0 ]]; then
      exit 0
    elif [[ $audit_result -eq 10 ]]; then
      exit 10
    else
      echo "ERROR: Milestone audit encountered an error" >&2
      exit 1
    fi
  fi
fi
```

This ensures the audit is also triggered when autopilot is restarted after all phases are already marked complete (e.g., resuming after a crash, or running `--from-phase` on an already-complete milestone). Without this, the startup exit would bypass the audit entirely.
