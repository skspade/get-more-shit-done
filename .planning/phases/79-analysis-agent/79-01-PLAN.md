**`<process>` section — 6 steps:**

**Step 1: Parse Diff Input**
1. Read all files from `<files_to_read>` block
2. Parse `<test-review-input>` for mode, diff base, changed files, test count, test config
3. Categorize each changed file:
   - **New files:** Added in the diff (no corresponding `--- a/` path, or `--- /dev/null`)
   - **Modified files:** Changed content in existing files
   - **Deleted files:** Removed in the diff (`+++ /dev/null`)
4. Separate source files from test files in the changed list (test files match `*.test.*` or `*.spec.*`)
5. Extract budget from test config: `budget.project` (default 800)
6. Calculate budget status: OK (<80%), Warning (80-99%), Over Budget (>=100%)

**Step 2: Source-to-Test Mapping (AGT-01)**

For each changed source file (not test files, not config/docs):

Signal 1 — Naming conventions:
- Look for `{basename}.test.{ext}` and `{basename}.spec.{ext}` in the same directory and `__tests__/` subdirectory
- Use Glob tool to find matches

Signal 2 — Import tracing:
- Use Grep tool to search all test files for `require` or `import` statements that reference the changed source module path
- Pattern: search for the source file's module path (without extension) in test files

Record mapping: `{source_file} -> [{test_file_1}, {test_file_2}, ...]`
Record unmapped files (source files with NO test file found via either signal)

**Step 3: Coverage Gap Detection (AGT-02, AGT-03)**

3a. Missing test files (AGT-03):
- From the mapping in Step 2, list source files with zero test file matches
- These are "missing test file" findings

3b. Untested exports (AGT-02):
- For new/modified source files that DO have test files:
  - Parse the diff hunks for lines containing `export`, `module.exports`, or function definitions
  - Extract function/variable names being exported
  - Use Grep to search the related test files for references to those function names
  - Functions with no test reference are "coverage gap" findings

**Step 4: Staleness Detection (AGT-04)**

For deleted and modified source files:
- Parse diff hunks for REMOVED lines (lines starting with `-`) that contain function definitions (`function`, `const`, `class`, or `export`)
- Extract the names of removed/renamed functions
- For each removed function name, search related test files (from Step 2 mapping) for references
- Test assertions referencing removed functions are "stale test" findings

**Step 5: Consolidation Analysis (AGT-05)**

Scope: Only test files related to the diff (from Step 2 mapping).

Apply four strategies:

**Prune:** Test files where ALL referenced source functions have been removed (fully stale). Recommendation: delete the test file.

**Parameterize:** Within a single test file related to the diff, look for multiple `it`/`test` blocks with similar descriptions testing the same function with different inputs. Recommendation: combine into `test.each` or data-driven pattern.

**Promote:** If both unit tests and integration tests exist for the same changed module, and the integration test covers the same function calls, recommend keeping integration test and removing redundant unit tests.

**Merge:** If >3 test files map to the same changed source file, recommend consolidating into fewer files.

Each finding includes: strategy name, source location, action, rationale, estimated test count impact.
