# Plan 01: Create /gsd:ui-test Command

## Objective

Create the `/gsd:ui-test` command file that parses arguments and flags, displays a GSD banner, spawns the `gsd-playwright` agent via `Task()`, and presents structured results.

## must_haves

- [ ] Command accepts phase number, URL, free-text instructions, and flags (`--scaffold`, `--run-only`, `--headed`)
- [ ] Command spawns `gsd-playwright` agent with correct `<playwright_input>` block
- [ ] Command displays GSD banner format on start and completion
- [ ] `--scaffold` and `--run-only` are mutually exclusive with clear error
- [ ] Agent results are parsed and displayed with pass/fail/skipped counts

## Tasks

<task id="1" title="Create command file with YAML frontmatter and argument parsing">

Create `commands/gsd/ui-test.md` following the `audit-tests.md` structural pattern.

**YAML frontmatter:**
```yaml
---
name: gsd:ui-test
description: Generate and run Playwright E2E tests against your application
argument-hint: "[phase] [url] [--scaffold] [--run-only] [--headed]"
allowed-tools:
  - Read
  - Glob
  - Grep
  - Bash
  - Task
---
```

**`<objective>` section:**
Spawn the gsd-playwright agent to generate and run Playwright E2E tests. This command IS the orchestrator — it parses arguments, gathers pre-flight data, spawns the agent, and presents results.

**`<execution_context>` section:**
```
No workflow file needed -- this is a direct agent spawn.
```

**`<context>` section:**
Document the argument parsing:
- Phase number: first numeric token from `$ARGUMENTS` (optional) -> `$PHASE_ARG`
- URL: token matching `http://` or `https://` (defaults to `http://localhost:3000`) -> `$BASE_URL`
- `--scaffold` flag -> `$FORCE_SCAFFOLD` (true/false)
- `--run-only` flag -> `$RUN_ONLY` (true/false)
- `--headed` flag -> `$HEADED_MODE` (true/false)
- Remaining tokens -> `$INSTRUCTIONS` (free-text passed to agent)

**`<process>` section — Step 1: Parse Arguments:**
Parse `$ARGUMENTS` for all tokens. Error if both `--scaffold` and `--run-only` are present:
```
Error: --scaffold and --run-only are mutually exclusive.
--scaffold forces Playwright setup; --run-only skips generation and runs existing tests.
```

**Step 2: Resolve Phase (if provided):**
When `$PHASE_ARG` is present:
```bash
PHASE_INFO=$(node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" init phase-op ${PHASE_ARG} 2>/dev/null)
```
Extract `phase_dir`, `phase_number`, `phase_name` from JSON. Error if phase not found.
When `$PHASE_ARG` is omitted, set `$PHASE_DIR` to null.

**Step 3: Detect Playwright State:**
```bash
DETECT=$(node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" playwright-detect --raw 2>/dev/null || echo '{"status":"not-detected"}')
```
Extract `status` from JSON for display.

**Step 4: Display Banner:**
When phase is provided:
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 GSD ► UI TEST — Phase {N}: {name}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```
When no phase:
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 GSD ► UI TEST
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

Display context lines:
- Playwright: `{status}` (from detection)
- URL: `{base_url}`
- Mode: `{scaffold | run-only | generate & run}` (derived from flags)
- Then: `◆ Spawning playwright agent...`

**Step 5: Resolve Model and Spawn Agent:**
```bash
PLAYWRIGHT_MODEL=$(node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" resolve-model gsd-playwright --raw 2>/dev/null || echo "sonnet")
```

Determine mode:
- If `--scaffold`: mode is `scaffold`
- If `--run-only`: mode is `ui-test` (with `--run-only` in flags)
- Otherwise: mode is `ui-test`

Build flags string from active flags (`--scaffold`, `--run-only`, `--headed`).

Spawn via Task():
```
Task(
  prompt="First, read agents/gsd-playwright.md for your role and instructions.

<files_to_read>
- ./CLAUDE.md (if exists -- project instructions)
{- {phase_dir}/*-CONTEXT.md (if phase provided)}
</files_to_read>

<playwright_input>
**Mode:** {mode}
**Phase Dir:** {phase_dir or 'null'}
**Base URL:** {base_url}
**Flags:** {flags string}
{**Instructions:** {free-text instructions} (if provided)}
</playwright_input>

Generate and execute Playwright E2E tests.",
  subagent_type="gsd-playwright",
  model="{PLAYWRIGHT_MODEL}",
  description="Playwright E2E: Phase {phase_number}"
)
```

**Step 6: Parse Agent Return and Display Results:**

Parse the agent's return for `## PLAYWRIGHT COMPLETE` or `## PLAYWRIGHT BLOCKED`.

On `## PLAYWRIGHT COMPLETE`:
- Extract Status (GREEN/RED/BLOCKED), Results line, Failure Details table
- GREEN: display pass/fail/skipped counts with completion banner
- RED: display counts plus failure details table
- BLOCKED: display reason

Completion banners:
- GREEN: `GSD ► UI TEST COMPLETE ✓`
- RED: `GSD ► UI TEST FAILED ✗` with failure summary
- BLOCKED: `GSD ► UI TEST BLOCKED` with reason

On `## PLAYWRIGHT BLOCKED`:
- Display blocker reason with error banner

</task>
