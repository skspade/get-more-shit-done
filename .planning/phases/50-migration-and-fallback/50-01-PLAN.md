# Plan 50-01: Dependency, Legacy Rename, and Test Retirement

## Objective

Add `zx` as a runtime dependency in `package.json`, rename `autopilot.sh` to `autopilot-legacy.sh` via `git mv`, and delete the now-unnecessary `format-json-output.test.cjs` test file. These are the preparatory changes needed before the entrypoint can be rewritten.

## must_haves

- `zx` appears in `package.json` `dependencies` (not `devDependencies`) with a caret major version pin (e.g., `^8.0.0`)
- `get-shit-done/scripts/autopilot.sh` no longer exists at the original path
- `get-shit-done/scripts/autopilot-legacy.sh` exists with identical content to what `autopilot.sh` had
- `tests/format-json-output.test.cjs` is deleted
- No other files are modified beyond what is listed

## Tasks

<task id="1">
<title>Add zx to package.json dependencies</title>
<actions>
1. Read `package.json`
2. Check the currently installed zx version: run `npm show zx version` to get the latest major version
3. Add `"zx": "^8.0.0"` (or appropriate major version from the check) to the `dependencies` field in `package.json`. If no `dependencies` field exists, create it after `devDependencies`
4. Run `npm install` to update `package-lock.json` (if lock file exists) and validate the dependency resolves
5. Verify: `node -e "const pkg = require('./package.json'); console.log(pkg.dependencies.zx)"` prints the version range
</actions>
</task>

<task id="2">
<title>Rename autopilot.sh to autopilot-legacy.sh</title>
<actions>
1. Run `git mv get-shit-done/scripts/autopilot.sh get-shit-done/scripts/autopilot-legacy.sh`
2. Verify the original path no longer exists: `test ! -f get-shit-done/scripts/autopilot.sh && echo "OK"`
3. Verify the new path exists: `test -f get-shit-done/scripts/autopilot-legacy.sh && echo "OK"`
4. Verify content is intact: `head -1 get-shit-done/scripts/autopilot-legacy.sh` should show the shebang line
</actions>
</task>

<task id="3">
<title>Delete format-json-output.test.cjs</title>
<actions>
1. Run `git rm tests/format-json-output.test.cjs`
2. Verify: `test ! -f tests/format-json-output.test.cjs && echo "OK"`
3. Run the test suite to confirm no test runner errors from the missing file: `npm test` — verify it passes (the test runner should not reference the deleted file explicitly since it discovers tests via glob)
</actions>
</task>
