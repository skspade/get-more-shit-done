Insert a new **Step 3: Interview** between the current Step 2 (fetch issue data) and current Step 3 (route via heuristic). Do NOT renumber or modify other steps yet — Plan 02 handles that.

The new Step 3 has two parts:

**Part A: Pre-scan ticket data**

After the Step 2 banner and issue display, add pre-scan logic:

1. Read `$ISSUES[0]` (primary issue) fields: title, description, labels, comments
2. Build a skip checklist with these conservative rules:
   - `$SKIP_GOAL = true` ONLY if description contains a markdown section `## Goal` or `## Objective` with content
   - `$SKIP_SCOPE = true` ONLY if description explicitly names files/components OR has a `## Scope` section
   - `$SKIP_CRITERIA = true` ONLY if description contains `## Acceptance Criteria` or `## Done When` section with items
   - `$SKIP_COMPLEXITY = true` if `$FORCE_QUICK` or `$FORCE_MILESTONE` is true (from Step 1 flag parsing)
   - If `$SKIP_COMPLEXITY` is false, also check: if description or labels explicitly state "quick fix", "small change", "epic", or "milestone" — set `$SKIP_COMPLEXITY = true` and store `$INFERRED_COMPLEXITY` with the inferred value
   - `$SKIP_ADDITIONAL` starts false (asked only if previous answers surface ambiguity)

Display pre-scan results:
```
◆ Pre-scan: {N} of 5 questions will be skipped (ticket provides answers)
```

**Part B: Adaptive interview questions**

Ask questions one at a time using AskUserQuestion, following the brainstorm.md adaptive pattern. Initialize `$INTERVIEW_CONTEXT = ""`.

**Q1: Goal clarification** (skip if `$SKIP_GOAL`)
- If skipped: extract goal from ticket's ## Goal section, append to `$INTERVIEW_CONTEXT`: `**Goal:** (from ticket) {extracted}`
- If asked:
```
AskUserQuestion(
  header: "Interview: Goal",
  question: "What's the core outcome you want from {$ISSUES[0].identifier}?",
  options: [generate 2-3 interpretations from title if ambiguous, plus "Something else — let me describe"],
  followUp: null
)
```
Append answer to `$INTERVIEW_CONTEXT`: `**Goal:** {answer}`

**Q2: Scope boundaries** (skip if `$SKIP_SCOPE`)
- If skipped: extract scope from ticket sections/file references, append: `**Scope:** (from ticket) {extracted}`
- If asked: Use options derived from ticket context (component names, areas). Include "Broader — touches multiple areas" and "Not sure yet" options.
Append: `**Scope:** {answer}`

**Q3: Success criteria** (skip if `$SKIP_CRITERIA`)
- If skipped: extract criteria from ticket's acceptance criteria section, append: `**Success Criteria:** (from ticket) {extracted}`
- If asked: Offer options synthesized from the goal answer (from Q1). Include "I'll define these after seeing the approach" option.
Append: `**Success Criteria:** {answer}`

**Q4: Complexity signal** (skip if `$SKIP_COMPLEXITY`)
- If skipped via flag: append: `**Complexity:** (flag override) {$FORCE_QUICK ? "Quick" : "Milestone"}`
- If skipped via ticket inference: append: `**Complexity:** (from ticket) {$INFERRED_COMPLEXITY}` — but do NOT set route yet (Plan 02 handles routing)
- If asked:
```
AskUserQuestion(
  header: "Interview: Complexity",
  question: "How much work does this feel like?",
  options: ["Quick task (hours)", "Medium (1-2 sessions)", "Milestone (multi-phase)"],
  followUp: null
)
```
Append: `**Complexity:** {answer}`

**Q5: Additional context** (conditional — only ask if previous answers surfaced ambiguity)
- Ambiguity signals: Q1 answer was "Something else", Q2 answer was "Not sure yet", Q3 was deferred, or any answer contradicts ticket data.
- If asked:
```
AskUserQuestion(
  header: "Interview: Additional",
  question: "Anything else I should know about this task?",
  followUp: null
)
```
Append: `**Additional:** {answer}`
- If not asked: Append: `**Additional:** none`

After all questions, display:
```
✓ Interview complete ({N} questions asked, {M} skipped from ticket)
```
