# Plan 18-02: Help Command

## Objective

Extend the existing `handleHelp` function in `cli.cjs` to support per-command detailed help (`gsd help <command>`) while preserving the existing overview behavior for `gsd help`.

## must_haves

- `gsd help` displays all available commands with one-line descriptions (existing behavior preserved)
- `gsd help <command>` displays detailed usage, flags, and examples for that specific command
- `gsd help <unknown>` shows an error listing available commands
- JSON and plain output modes work correctly via `formatOutput()`
- All existing tests continue to pass

## Verification

After implementation, run:
```bash
node tests/cli.test.cjs
```
All tests must pass with zero failures.

Manual verification:
```bash
node get-shit-done/bin/gsd-cli.cjs help
node get-shit-done/bin/gsd-cli.cjs help progress
node get-shit-done/bin/gsd-cli.cjs help nonexistent
node get-shit-done/bin/gsd-cli.cjs help --json
node get-shit-done/bin/gsd-cli.cjs help progress --json
```

<task id="1" title="Add COMMAND_DETAILS registry and extend handleHelp">

**What:** Add a `COMMAND_DETAILS` static registry object containing per-command usage, description, flags, and examples. Extend `handleHelp` to look up per-command details when `args[0]` is provided.

**Where:** `get-shit-done/bin/lib/cli.cjs`

**How:**

1. Add a `COMMAND_DETAILS` object near the `COMMANDS` registry with entries for each command:

```javascript
const COMMAND_DETAILS = {
  progress: {
    usage: 'gsd progress [--json] [--plain]',
    description: 'Show milestone progress and status. Displays the current milestone name, version, phase list with completion status, plan counts, progress bar, and suggested next action.',
    flags: [
      { flag: '--json', description: 'Output as JSON' },
      { flag: '--plain', description: 'Output as plain text (no colors)' },
    ],
    examples: [
      'gsd progress',
      'gsd progress --json',
    ],
  },
  todos: {
    usage: 'gsd todos [<id>] [--area=<area>] [--json] [--plain]',
    description: 'List and inspect pending todos. Shows all pending todos by default. Use an ID to see full details of a specific todo, or filter by area.',
    flags: [
      { flag: '<id>', description: 'Show full details for a specific todo' },
      { flag: '--area=<area>', description: 'Filter todos by area' },
      { flag: '--json', description: 'Output as JSON' },
      { flag: '--plain', description: 'Output as plain text (no colors)' },
    ],
    examples: [
      'gsd todos',
      'gsd todos fix-login-bug',
      'gsd todos --area=bugfix',
    ],
  },
  health: {
    usage: 'gsd health [--json] [--plain]',
    description: 'Validate .planning/ directory integrity. Checks that required files exist, config.json is valid, and STATE.md is consistent with ROADMAP.md.',
    flags: [
      { flag: '--json', description: 'Output as JSON' },
      { flag: '--plain', description: 'Output as plain text (no colors)' },
    ],
    examples: [
      'gsd health',
      'gsd health --json',
    ],
  },
  settings: {
    usage: 'gsd settings [set <key> <value>] [--json] [--plain]',
    description: 'View and update configuration. Shows all config.json settings by default. Use "set" to update a value with validation.',
    flags: [
      { flag: 'set <key> <value>', description: 'Set a config value (dot notation for nested keys)' },
      { flag: '--json', description: 'Output as JSON' },
      { flag: '--plain', description: 'Output as plain text (no colors)' },
    ],
    examples: [
      'gsd settings',
      'gsd settings set model_profile quality',
      'gsd settings set workflow.research false',
    ],
  },
  help: {
    usage: 'gsd help [<command>]',
    description: 'Show available commands and usage. Without arguments, lists all commands. With a command name, shows detailed usage information.',
    flags: [
      { flag: '<command>', description: 'Show detailed help for a specific command' },
    ],
    examples: [
      'gsd help',
      'gsd help progress',
    ],
  },
};
```

2. Modify `handleHelp()` to accept `(projectRoot, args)`:
   - If `args && args.length > 0`: per-command detail mode
     - Look up `args[0]` in `COMMAND_DETAILS`
     - If not found: return `{ command: 'help', error: true, message: "Unknown command: '<args[0]>'\n\nAvailable commands: progress, todos, health, settings, help" }`
     - If found: build rich output with:
       - Usage line in bold
       - Blank line
       - Description paragraph
       - Blank line
       - Flags/arguments section header in bold, then list each flag with `  {flag}  {DIM}{description}{RESET}`
       - Blank line
       - Examples section header in bold, then list each example with `  {DIM}{example}{RESET}`
     - Return `{ command: 'help', detail: { name: args[0], ...COMMAND_DETAILS[args[0]] }, message }`
   - If no args: existing overview behavior (unchanged logic, same output)

**ANSI constants** to add inside `handleHelp`: `BOLD`, `RESET`, `DIM` (same as other handlers).

</task>

<task id="2" title="Add tests for help command per-command detail">

**What:** Add tests for the new per-command help detail feature and verify existing help overview behavior is preserved.

**Where:** `tests/cli.test.cjs`

**How:**

1. Add a new `describe('handleHelp', ...)` block after the settings tests:

   **Tests:**

   a. `'overview mode lists all commands with descriptions (HELP-01)'`
      - Call `routeCommand('help', null, [], 'json')`
      - Assert `result.command === 'help'`
      - Assert `result.message` includes `'progress'`, `'todos'`, `'health'`, `'settings'`, `'help'`
      - Assert `result.message` includes `'Usage:'`

   b. `'per-command detail returns usage, description, flags, examples (HELP-02)'`
      - Call `routeCommand('help', null, ['progress'], 'json')`
      - Assert `result.command === 'help'`
      - Assert `result.detail` exists
      - Assert `result.detail.name === 'progress'`
      - Assert `result.detail.usage` includes `'gsd progress'`
      - Assert `result.detail.description` is a non-empty string
      - Assert `result.detail.flags` is an array with length > 0
      - Assert `result.detail.examples` is an array with length > 0

   c. `'per-command detail works for all commands (HELP-02)'`
      - For each of `['progress', 'todos', 'health', 'settings', 'help']`:
        - Call `routeCommand('help', null, [cmd], 'json')`
        - Assert `result.detail` exists
        - Assert `result.detail.usage` is a non-empty string

   d. `'unknown command returns error with available commands (HELP-02)'`
      - Call `routeCommand('help', null, ['nonexistent'], 'json')`
      - Assert `result.error === true`
      - Assert `result.message` includes `'nonexistent'`
      - Assert `result.message` includes `'progress'` (lists available commands)

   e. `'per-command detail rich mode includes ANSI formatting'`
      - Call `routeCommand('help', null, ['todos'], 'rich')`
      - Assert `result.message` includes `'\x1b['` (ANSI codes)
      - Assert `result.message` includes `'gsd todos'`

   f. `'help --json returns structured data for overview'`
      - Call `routeCommand('help', null, [], 'json')`
      - Assert `result.command === 'help'`
      - Assert `typeof result.message === 'string'`

2. Add integration test in the `gsd-cli binary` describe block:

   g. `'help progress shows detailed usage'`
      - Run `node "${cliPath}" help progress` with `cwd: projectRoot`
      - Assert output includes `'gsd progress'`
      - Assert output includes `'--json'`

   h. `'help nonexistent shows error'`
      - Run `node "${cliPath}" help nonexistent` with `cwd: projectRoot`
      - Assert output includes `'nonexistent'` (note: help does not require projectRoot, so this returns exit code 0 with the error in output)

</task>
