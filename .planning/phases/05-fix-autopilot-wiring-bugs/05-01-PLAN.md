**Fix 1: `extract_verification_status` (line 673)** — Replace the single find with a dual-pattern search:
```bash
extract_verification_status() {
  local phase_dir="$1"
  local verification_file

  # Check for VERIFICATION.md first (automated verify-phase)
  verification_file=$(find "$PROJECT_DIR/$phase_dir" -name "*-VERIFICATION.md" -type f 2>/dev/null | head -1)

  # Fall back to UAT.md (interactive verify-work)
  if [[ -z "$verification_file" || ! -f "$verification_file" ]]; then
    verification_file=$(find "$PROJECT_DIR/$phase_dir" -name "*-UAT.md" -type f 2>/dev/null | head -1)
  fi

  if [[ -z "$verification_file" || ! -f "$verification_file" ]]; then
    echo "status=unknown|score=N/A"
    return 1
  fi

  local status score
  status=$(node "$GSD_TOOLS" frontmatter get "$verification_file" --field status 2>/dev/null || echo "unknown")
  score=$(node "$GSD_TOOLS" frontmatter get "$verification_file" --field score 2>/dev/null || echo "N/A")

  echo "status=${status}|score=${score}"
}
```

**Fix 2: `extract_gaps_summary` (line 690)** — Same dual-pattern search:
```bash
extract_gaps_summary() {
  local phase_dir="$1"
  local verification_file

  # Check VERIFICATION.md first, then UAT.md
  verification_file=$(find "$PROJECT_DIR/$phase_dir" -name "*-VERIFICATION.md" -type f 2>/dev/null | head -1)
  if [[ -z "$verification_file" || ! -f "$verification_file" ]]; then
    verification_file=$(find "$PROJECT_DIR/$phase_dir" -name "*-UAT.md" -type f 2>/dev/null | head -1)
  fi

  [[ -z "$verification_file" || ! -f "$verification_file" ]] && return 0

  # Extract lines from gaps-related sections
  sed -n '/^## .*[Gg]ap/,/^## [^G]/p' "$verification_file" | grep -E "^[-*]|FAILED|MISSING|STUB|NOT_WIRED" | head -10 || true
}
```

**Why VERIFICATION.md is preferred over UAT.md:** VERIFICATION.md comes from the automated verify-phase (execute-phase internal check). UAT.md comes from interactive verify-work. The automated check is more authoritative. If both exist in the same phase dir, use VERIFICATION.md.

**What this fixes for FAIL-02:** `run_verify_with_debug_retry` calls `extract_verification_status` to check if gaps were found. Previously, when verify-work wrote a UAT.md with `status: gaps_found`, the function returned `status=unknown` (which is NOT `gaps_found`), so the retry loop treated it as passed. Now it correctly reads the UAT.md and returns the actual status, enabling debug-retry to detect gaps and attempt fixes.
