**Workflow** (`get-shit-done/workflows/pr-review.md`):
Create following the linear.md workflow pattern:

`<purpose>` block: Describe parsing args, capturing review (fresh or ingest), extracting findings.

`<process>` block with these steps:

**Step 1: Parse arguments**
Parse `$ARGUMENTS` for:
- `--ingest` flag -> `$INGEST_MODE` (true/false)
- `--quick` flag -> `$FORCE_QUICK` (true/false)
- `--milestone` flag -> `$FORCE_MILESTONE` (true/false)
- `--full` flag -> `$FULL_MODE` (true/false)
- Remaining tokens -> `$REVIEW_ASPECTS` (array of aspect strings)

If both `--quick` and `--milestone` present: Error — "Cannot use both --quick and --milestone flags."

**Step 2: Capture review**

*Fresh mode* (when `$INGEST_MODE` is false):
- Invoke `/pr-review-toolkit:review-pr` via the Skill tool, passing `$REVIEW_ASPECTS` as aspect arguments
- Capture the aggregated output as `$RAW_REVIEW`

*Ingest mode* (when `$INGEST_MODE` is true):
- Use AskUserQuestion with header "PR Review Ingest", question "Paste or provide the PR review summary:"
- Store response as `$RAW_REVIEW`

**Step 3: Parse findings**
Parse `$RAW_REVIEW` into structured findings array. For each finding, extract:
- `severity`: Map from section headers — `## Critical Issues` -> "critical", `## Important Issues` -> "important", `## Suggestions` -> "suggestion"
- `agent`: Extract from `[agent-name]:` prefix pattern in finding text
- `description`: The finding text (after agent prefix, before file reference)
- `file`: Extract from `[file:line]` or file path reference pattern, null if absent
- `line`: Extract line number from file reference, null if absent
- `fix_suggestion`: Extract suggested fix text if present, null if absent

If no findings parsed: Display "No actionable issues found." and exit workflow cleanly (no error).
